2.1 Data Manipulation Language (DML)

DML statements are used to add, change, or delete data within database tables.

INSERT INTO

Adds new rows of data into a table.

-- Insert a full row
INSERT INTO Customers (CustomerID, CustomerName, ContactName, City, Country)
VALUES (5, 'New Customer', 'John Doe', 'Nairobi', 'Kenya');

-- Insert into specific columns
INSERT INTO Customers (CustomerName, City)
VALUES ('Jane Smith', 'Mombasa');
UPDATE

Modifies existing records in a table. Always use a WHERE clause to avoid updating all rows!

-- Update a specific record
UPDATE Customers
SET City = 'Kisumu', Country = 'Kenya'
WHERE CustomerID = 2;

-- Update multiple records
UPDATE Orders
SET Status = 'Delivered'
WHERE Status = 'Shipped' AND OrderDate < '2023-03-01';
DELETE

Removes existing records from a table. Like UPDATE, always use a WHERE clause.

-- Delete a specific record
DELETE FROM Customers
WHERE CustomerID = 5;

-- Delete based on condition
DELETE FROM Orders
WHERE Status = 'Cancelled';
⚠️ Critical Warning: Running UPDATE or DELETE without a WHERE clause will modify or erase ALL records in the table!

2.2 Aggregate Functions

Aggregate functions perform a calculation on a set of values and return a single value.

FunctionDescriptionExample
COUNT()Returns the number of rowsSELECT COUNT(CustomerID) FROM Customers;
SUM()Returns the total sum of a numeric columnSELECT SUM(TotalAmount) FROM Orders;
AVG()Returns the average valueSELECT AVG(TotalAmount) FROM Orders;
MIN()Returns the smallest valueSELECT MIN(TotalAmount) FROM Orders;
MAX()Returns the largest valueSELECT MAX(TotalAmount) FROM Orders;
 
-- Count customers in Germany
SELECT COUNT(CustomerID) AS GermanCustomers
FROM Customers
WHERE Country = 'Germany';

-- Calculate total and average sales
SELECT 
    SUM(TotalAmount) AS TotalSales,
    AVG(TotalAmount) AS AverageOrderValue,
    MIN(TotalAmount) AS SmallestOrder,
    MAX(TotalAmount) AS LargestOrder
FROM Orders;

2.3 Grouping Data

GROUP BY

Groups rows that have the same values into summary rows. It is often used with aggregate functions.

-- Count customers per country
SELECT Country, COUNT(CustomerID) AS CustomerCount
FROM Customers
GROUP BY Country;

-- Calculate total sales per status
SELECT Status, SUM(TotalAmount) AS TotalSales
FROM Orders
GROUP BY Status;
HAVING

The HAVING clause was added to SQL because the WHERE keyword could not be used with aggregate functions. It filters groups.

-- Find countries with more than 2 customers
SELECT Country, COUNT(CustomerID) AS CustomerCount
FROM Customers
GROUP BY Country
HAVING COUNT(CustomerID) > 2;

-- Find statuses with total sales over 500
SELECT Status, SUM(TotalAmount) AS TotalSales
FROM Orders
GROUP BY Status
HAVING SUM(TotalAmount) > 500;
WHERE vs. HAVING: WHERE filters rows before they are grouped. HAVING filters groups after the aggregation is performed.

2.4 Joining Tables

Joins are used to combine rows from two or more tables based on a related column between them.

INNER JOIN

Returns records that have matching values in both tables.

SELECT Customers.CustomerName, Orders.OrderID
FROM Customers
INNER JOIN Orders ON Customers.CustomerID = Orders.CustomerID;
LEFT (OUTER) JOIN

Returns all records from the left table, and the matched records from the right table. Returns NULL for right side if no match.

SELECT Customers.CustomerName, Orders.OrderID
FROM Customers
LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;
RIGHT (OUTER) JOIN

Returns all records from the right table, and the matched records from the left table.

SELECT Customers.CustomerName, Orders.OrderID
FROM Customers
RIGHT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;
FULL (OUTER) JOIN

Returns all records when there is a match in either left or right table.

SELECT Customers.CustomerName, Orders.OrderID
FROM Customers
FULL OUTER JOIN Orders ON Customers.CustomerID = Orders.CustomerID;
SELF JOIN

A table is joined with itself. Useful for finding hierarchical data or comparing rows within the same table.

-- Find customers in the same city
SELECT A.CustomerName AS Customer1, B.CustomerName AS Customer2, A.City
FROM Customers A, Customers B
WHERE A.CustomerID <> B.CustomerID
AND A.City = B.City;

2.5 Combining Queries

UNION

Combines the result-set of two or more SELECT statements. Each statement must have the same number of columns with similar data types. UNION removes duplicate rows.

SELECT City FROM Customers
UNION
SELECT City FROM Suppliers; -- Assuming a Suppliers table exists
UNION ALL

Combines the result-sets but allows duplicate rows. It is faster than UNION because it doesn't sort and remove duplicates.

SELECT City FROM Customers
UNION ALL
SELECT City FROM Suppliers;

2.6 Advanced Querying

EXISTS

Tests for the existence of any record in a subquery. Returns true if the subquery returns one or more records.

-- Find customers who have placed at least one order
SELECT CustomerName
FROM Customers
WHERE EXISTS (
    SELECT 1 FROM Orders 
    WHERE Orders.CustomerID = Customers.CustomerID
);
ANY and ALL

Used with comparison operators and subqueries. ANY is true if the comparison is true for any value in the subquery. ALL is true if it's true for all values.

-- Find customers who have orders greater than ANY order from CustomerID = 1
SELECT CustomerName
FROM Customers
WHERE CustomerID = ANY (
    SELECT CustomerID FROM Orders WHERE TotalAmount > 200
);
CASE Statement

Creates conditional logic within a query, similar to an IF-THEN-ELSE statement in programming.

SELECT OrderID, TotalAmount,
    CASE 
        WHEN TotalAmount > 300 THEN 'High Value'
        WHEN TotalAmount > 100 THEN 'Medium Value'
        ELSE 'Low Value'
    END AS OrderCategory
FROM Orders;
NULL Functions

Functions to handle NULL values gracefully.

-- COALESCE: Returns the first non-null value
SELECT CustomerName, COALESCE(ContactName, 'Unknown Contact') AS Contact
FROM Customers;

-- IFNULL (MySQL) / ISNULL (SQL Server): Replaces NULL with a specified value
SELECT CustomerName, IFNULL(City, 'No City Provided') AS City
FROM Customers;
