Foundations of Statistical Inference

Hypothesis testing is the backbone of inferential statistics. It allows us to make decisions about a population based on sample data, quantifying the uncertainty of our conclusions.

The Framework
  • 1. Null Hypothesis ($H_0$): The default assumption (e.g., "no effect", "no difference"). We assume this is true until proven otherwise.
  • 2. Alternative Hypothesis ($H_1$ or $H_a$): The claim we are trying to find evidence for (e.g., "treatment works", "means differ").
  • 3. P-value: The probability of observing data as extreme as ours, assuming $H_0$ is true.
    Rule of thumb: If $p \le \alpha$ (usually 0.05), reject $H_0$.
  • 4. Errors:
    Type I ($\alpha$): Rejecting a true $H_0$ (False Positive).
    Type II ($\beta$): Failing to reject a false $H_0$ (False Negative).
Choosing the Right Test

The choice of test depends on your data types and research question:

Dependent VariableIndependent VariableTest
ContinuousNone (Compare to constant)One-Sample T-Test
ContinuousCategorical (2 groups)Two-Sample T-Test
ContinuousCategorical (3+ groups)ANOVA
CategoricalNone (Compare to dist.)Chi-Square Goodness of Fit
CategoricalCategoricalChi-Square Independence

T-Tests: Comparing Means

The T-test is used to determine if there is a statistically significant difference between the means of two groups. It is ideal for small sample sizes and unknown population variances.

1. One-Sample T-Test

Concept & Relevance

Tests whether the mean of a single sample differs from a known standard, population mean, or theoretical value.

$$ t = \frac{\bar{x} - \mu_0}{s / \sqrt{n}} $$
Statistical Assumptions
  • Normality: The data should be approximately normally distributed (check via Shapiro-Wilk test or Q-Q plot).
  • Independence: Observations are independent of each other.
  • Scale: The dependent variable must be continuous (interval/ratio).
R Implementation
# Scenario: Testing if average battery life differs from 10 hours
battery_life <- c(9.8, 10.2, 9.5, 10.5, 10.1, 9.9, 10.3, 9.7)

# Perform one-sample t-test (mu = 10)
t_test_result <- t.test(battery_life, mu = 10)

print(t_test_result)
# Output interpretation: If p-value < 0.05, the mean is significantly different from 10.
Practical Application

Quality Control: A manufacturer claims their cereal boxes contain 500g. You sample 30 boxes to verify if the true mean weight is actually 500g or if the machine is under/over-filling.

2. Two-Sample (Unpaired/Independent) T-Test

Concept & Relevance

Compares the means of two independent (unrelated) groups to see if there is a significant difference between them.

$$ t = \frac{\bar{x}_1 - \bar{x}_2}{\sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}} $$
Statistical Assumptions
  • Independence: The two groups must be independent of each other.
  • Normality: Data in both groups should be normally distributed.
  • Homogeneity of Variance: Variances of the two groups should be equal (tested via Levene's or F-test).
    Note: If variances are unequal, use Welch's T-Test (default in R).
R Implementation
# Scenario: Comparing test scores of two teaching methods
group_A <- c(85, 90, 78, 88, 92)
group_B <- c(75, 80, 82, 79, 85)

# Standard Student's T-test (Assumes equal variance)
t.test(group_A, group_B, var.equal = TRUE)

# Welch's T-test (Does NOT assume equal variance - Recommended)
t.test(group_A, group_B, var.equal = FALSE)
Practical Application

A/B Testing in Marketing: Comparing the average click-through rates of a website with a red "Buy" button (Group A) versus a green "Buy" button (Group B) to see which color drives more engagement.

3. Paired T-Test

Concept & Relevance

Compares the means of two related groups. It is used when the same subjects are measured twice (e.g., before and after an intervention), or when subjects are perfectly matched.

$$ t = \frac{\bar{d}}{s_d / \sqrt{n}} $$ Where $\bar{d}$ is the mean of the differences.
Statistical Assumptions
  • Paired Observations: Data points in group 1 must be naturally linked to data points in group 2.
  • Normality of Differences: The differences between the pairs should be normally distributed (not necessarily the raw data).
  • No extreme outliers: Outliers in the differences can heavily skew the result.
R Implementation
# Scenario: Testing a new diet program (Weight Before vs After)
before <- c(85, 90, 78, 88, 92, 80)
after  <- c(82, 86, 79, 84, 88, 78) # Same individuals

# Perform paired t-test
t.test(before, after, paired = TRUE, alternative = "greater")
# alternative="greater" tests if 'before' is significantly greater than 'after'
Practical Application

Clinical Trials: Measuring patients' blood pressure before administering a new drug and again after 4 weeks of treatment to determine if the drug effectively lowers blood pressure within the same cohort.

Chi-Square ($\chi^2$) Tests: Categorical Data

While T-tests compare means of continuous data, Chi-Square tests analyze the distribution of categorical data, testing for goodness of fit or independence between variables.

1. Chi-Square Goodness of Fit

Concept: Tests if a sample distribution matches a expected population distribution.

$$ \chi^2 = \sum \frac{(O_i - E_i)^2}{E_i} $$ Where $O$ is Observed frequency and $E$ is Expected frequency.
Assumptions
  • Data consists of frequencies/categories (not percentages).
  • Categories are mutually exclusive.
  • Expected frequency in each category should be $\ge 5$.
Application

Genetics: Testing if the offspring of a dihybrid cross follow the expected 9:3:3:1 Mendelian ratio.

R Implementation
# Scenario: Testing if a 6-sided die is fair
# Observed rolls for 60 trials
observed <- c(8, 12, 10, 10, 8, 12) 

# Expected probabilities (1/6 for each side)
expected_prob <- rep(1/6, 6)

chisq.test(observed, p = expected_prob)

2. Chi-Square Test of Independence

Concept: Tests if there is a significant association between two categorical variables.

Assumptions
  • Both variables are categorical.
  • Independent observations (one data point per subject).
  • Expected cell counts $\ge 5$ (If violated, use Fisher's Exact Test).
Application

Market Research: Determining if gender (Male/Female) is independent of product preference (Brand A/Brand B/Brand C).

R Implementation
# Scenario: Relationship between Smoking and Exercise level
# Create a contingency table
survey_data <- data.frame(
  Smoking = c("Yes", "No", "Yes", "No"),
  Exercise = c("High", "High", "Low", "Low"),
  Count = c(10, 30, 20, 40)
)
contingency_table <- xtabs(Count ~ Smoking + Exercise, data = survey_data)

# Perform Chi-Square test
chisq.test(contingency_table)

# Check expected values to ensure they are >= 5
chisq.test(contingency_table)$expected

The ANOVA Family: Experimental Design

Analysis of Variance (ANOVA) generalizes the T-test to compare means across three or more groups. By partitioning the total variance into "between-group" and "within-group" components, we can determine if group means differ significantly.

$$ F = \frac{MS_{between}}{MS_{within}} = \frac{\text{Variance explained by groups}}{\text{Unexplained variance (error)}} $$

Analysis of Variance (ANOVA)

Relevance: The foundational tool for comparing 3+ groups. One-Way ANOVA uses one independent variable (factor), while Two-Way ANOVA uses two, allowing us to test for interaction effects.

Assumptions
  • Normality: Residuals are normally distributed.
  • Homogeneity of Variance: Variances across groups are equal (Tested via Levene's Test).
  • Independence: Observations are independent.
Application

Agriculture: Comparing the crop yield of four different types of fertilizer to see if any performs significantly better than the others.

R Implementation (One-Way)
# Scenario: Crop yield across 3 fertilizers
model_aov <- aov(Yield ~ Fertilizer, data = crop_data)
summary(model_aov)

# Post-hoc test (Tukey's HSD) to see WHICH groups differ
TukeyHSD(model_aov)
R Implementation (Two-Way)
# Testing Fertilizer AND Irrigation, plus their interaction
model_2way <- aov(Yield ~ Fertilizer * Irrigation, data = crop_data)
summary(model_2way)

Analysis of Covariance (ANCOVA)

Relevance: ANCOVA is ANOVA + Regression. It allows you to compare group means while statistically controlling for a continuous "nuisance" variable (covariate) that might influence the outcome. This increases statistical power by reducing error variance.

Assumptions
  • All standard ANOVA assumptions apply.
  • Homogeneity of Regression Slopes: The relationship between the covariate and the dependent variable must be the same across all groups (no interaction between covariate and the independent variable).
  • The covariate is measured without error.
Application

Education: Comparing the final exam scores of three teaching methods (IV), while controlling for students' baseline IQ scores (Covariate) to ensure initial intelligence doesn't bias the results.

R Implementation
# Scenario: Final Score ~ Teaching Method + Baseline IQ
model_ancova <- aov(Final_Score ~ Teaching_Method + Baseline_IQ, 
                    data = student_data)
summary(model_ancova)

# Check assumption: Homogeneity of slopes
# (Test interaction between Method and Covariate)
model_check <- aov(Final_Score ~ Teaching_Method * Baseline_IQ, 
                   data = student_data)
summary(model_check) 
# If interaction p > 0.05, assumption is met.

Multivariate Analysis of Variance (MANOVA)

Relevance: When an experiment has two or more continuous dependent variables, running multiple ANOVAs increases the Type I error rate. MANOVA tests them simultaneously, accounting for the correlations between the dependent variables.

Assumptions
  • Multivariate Normality: The dependent variables follow a multivariate normal distribution.
  • Homogeneity of Covariance Matrices: The variance-covariance matrices are equal across groups (Tested via Box's M test). *MANOVA is robust to minor violations if sample sizes are equal.*
  • Linearity: Relationships between all pairs of DVs are linear.
Application

Medicine: Testing the effect of a new drug on three related health outcomes simultaneously: systolic blood pressure, diastolic blood pressure, and heart rate.

R Implementation
# Scenario: Effect of Diet on Weight and Cholesterol
# cbind() binds the multiple DVs together
model_manova <- manova(cbind(Weight, Cholesterol) ~ Diet, 
                       data = patient_data)
summary(model_manova)

# Use Pillai's Trace or Wilks' Lambda for the test statistic
summary(model_manova, test = "Pillai")

# Post-hoc for individual DVs
summary.aov(model_manova)

Multivariate Analysis of Covariance (MANCOVA)

Relevance: The most complex of the basic family. MANCOVA is MANOVA + Covariates. It tests for differences in multiple dependent variables across groups, while controlling for one or more continuous covariates.

Assumptions
  • All MANOVA assumptions apply.
  • All ANCOVA assumptions apply (specifically, homogeneity of regression slopes for all dependent variables).
  • Covariates are independent of the treatment (measured before treatment).
Application

Psychology: Evaluating the effect of three types of therapy (IV) on depression and anxiety scores (2x DVs), while controlling for the patients' age and baseline stress levels (Covariates).

R Implementation
# Scenario: Therapy effect on Depression & Anxiety, controlling for Age
model_mancova <- manova(
  cbind(Depression_Score, Anxiety_Score) ~ Therapy_Type + Age, 
  data = psych_data
)

summary(model_mancova, test = "Wilks")

# Note: MANCOVA requires careful checking of assumptions. 
# In modern practice, Linear Mixed Models (lme4) or 
# Structural Equation Modeling (SEM) are often preferred 
# for complex multivariate designs.

Advanced Experimental Designs

When experiments involve repeated measurements on the same subjects, or when logistical constraints prevent complete randomization, advanced designs are required to accurately model the error structure.

Repeated Measures ANOVA

Relevance: Used when the same subjects are measured three or more times under different conditions or over time. It accounts for the fact that measurements from the same person are correlated, removing "subject-to-subject" variance from the error term and drastically increasing statistical power.

Statistical Assumptions
  • Normality: The distribution of the dependent variable at each level should be normal.
  • Sphericity: The variances of the differences between all possible pairs of within-group conditions must be equal.
    Tested via Mauchly's Test of Sphericity.
    If violated ($p < 0.05$), apply Greenhouse-Geisser or Huynh-Feldt corrections.
  • No carryover effects (often managed by randomizing the order of treatments or adding a washout period).
Application

Ergonomics: Measuring the wrist strain of 20 workers using three different keyboard designs. Since it's the same workers, we use Repeated Measures to control for individual physical differences.

R Implementation
# Scenario: Strain measured at Time1, Time2, Time3 for each Subject
# Data must be in "Long Format"
# Columns: Subject, Time, Strain_Score

# Define the Error structure to account for repeated measures
model_rm <- aov(Strain_Score ~ Time + Error(Subject/Time), 
                data = ergonomics_data)

summary(model_rm)

# Pro-Tip: For missing data or unequal time intervals, 
# use Linear Mixed-Effects Models (lme4 package) instead:
# library(lme4)
# lmer(Strain_Score ~ Time + (1|Subject), data = ergonomics_data)

Split-Plot Design

Relevance: A highly practical design used when an experiment has two factors, but one factor (the "Whole-Plot" factor) is harder, more expensive, or logistically more difficult to change than the other (the "Sub-Plot" factor). It involves two levels of randomization.

Statistical Assumptions
  • Normality & Homogeneity: Standard assumptions apply to the residuals.
  • Two Error Terms: The design inherently has two sizes of experimental units. Therefore, there are two separate error terms: one for the Whole-Plot factor and one for the Sub-Plot factor (and their interaction).
  • Additivity of block and treatment effects.
Application

Agriculture:
Whole-Plot Factor: Irrigation method (requires large plots and expensive infrastructure to change).
Sub-Plot Factor: Fertilizer type (easy to apply to small sub-sections within the large irrigation plots).
The design ensures the logistical constraint doesn't ruin the statistical validity.

R Implementation
# Scenario: Irrigation (A) and Fertilizer (B) on Crop Yield
# 'Block' represents the replicates (fields)
# A is applied to whole plots, B to sub-plots

# Base R approach using Error() strata
model_split <- aov(Yield ~ A * B + Error(Block/A), 
                   data = ag_data)

summary(model_split)
# Note the output will show two sections: 
# Error: Block:A (Tests the Whole-plot factor A)
# Error: Within (Tests Sub-plot factor B and A:B interaction)

# Modern Approach: Linear Mixed Models (Recommended)
# library(lme4)
# lmer(Yield ~ A * B + (1|Block) + (1|Block:A), data = ag_data)
Note on Power: Because the whole-plot factor is applied to fewer, larger units, it has less statistical power (fewer degrees of freedom) than the sub-plot factor. Always assign the most important factor to the sub-plot if possible!
