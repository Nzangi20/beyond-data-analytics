What is Machine Learning?

Machine Learning (ML) is a subset of Artificial Intelligence that enables systems to learn and improve from experience without being explicitly programmed. It focuses on developing algorithms that can access data and use it to learn for themselves.

Historical Context

The term "Machine Learning" was coined by Arthur Samuel in 1959 while developing a checkers-playing program at IBM. He defined it as: "The field of study that gives computers the ability to learn without being explicitly programmed."

In 1997, Tom Mitchell provided the modern, widely-used definition: "A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E."

The Four Paradigms

Machine Learning is broadly categorized into four paradigms based on how the algorithm learns:

  • 1 Supervised Learning - Learning with labeled data
  • 2 Unsupervised Learning - Finding hidden patterns
  • 3 Semi-Supervised Learning - Small labeled + large unlabeled
  • 4 Reinforcement Learning - Learning through trial and reward
Supervised
Labeled data → Predict outcomes
Unsupervised
Unlabeled data → Find structure
Semi-Supervised
Mix of labeled & unlabeled
Reinforcement
Reward-based learning

1. Supervised Learning

Supervised learning algorithms learn a mapping function from input variables ($X$) to an output variable ($Y$) using labeled training data. The model is "supervised" because it is trained on known examples of inputs and their correct outputs.

Two Main Types:
  • Regression: Predicting a continuous numerical value (e.g., house price, temperature).
  • Classification: Predicting a discrete category/label (e.g., spam/not spam, species of flower).

Core Algorithms

Decision Trees (Classification & Regression)

Concept: A tree-like model of decisions. Each internal node represents a "test" on an attribute, each branch represents the outcome, and each leaf node represents a class label (classification) or a continuous value (regression).

History: The modern decision tree was developed in the 1960s by J. Ross Quinlan, who introduced the ID3 (Iterative Dichotomiser 3) algorithm in 1986, later improved to C4.5 and C5.0.

Assumptions & Characteristics
  • Non-parametric (no assumption about data distribution).
  • Handles both numerical and categorical data.
  • Prone to overfitting if not pruned.
  • Uses splitting criteria: Gini Impurity, Entropy/Information Gain, or Variance Reduction (for regression).
Applications
  • Medical diagnosis (symptom-based decision paths).
  • Credit scoring in banking.
  • Customer segmentation rules.
R Implementation (Classification with iris)
# install.packages("rpart")
# install.packages("rpart.plot")
library(rpart)
library(rpart.plot)

data(iris)

# Build classification tree
set.seed(42)
train_idx <- sample(1:nrow(iris), 0.7 * nrow(iris))
train_data <- iris[train_idx, ]
test_data  <- iris[-train_idx, ]

dt_model <- rpart(Species ~ ., data = train_data, 
                  method = "class",
                  control = rpart.control(cp = 0.01))

# Visualize the tree
rplot(dt_model, main = "Decision Tree - Iris Classification")

# Predictions
predictions <- predict(dt_model, test_data, type = "class")
table(Predicted = predictions, Actual = test_data$Species)

Random Forest

Concept: An ensemble method that builds many decision trees (using random subsets of data and features) and aggregates their predictions. For classification, it uses majority voting; for regression, it averages the outputs.

History: Introduced by Leo Breiman in 2001. It addresses the overfitting problem of single decision trees while maintaining high accuracy.

Assumptions & Characteristics
  • Handles high-dimensional data well.
  • Robust to outliers and noise.
  • Provides feature importance rankings.
  • Less interpretable than single trees.
  • Key hyperparameter: ntree (number of trees), mtry (features per split).
Applications
  • Gene expression analysis in bioinformatics.
  • Fraud detection in finance.
  • Image classification.
R Implementation (Classification with iris)
# install.packages("randomForest")
library(randomForest)

data(iris)
set.seed(42)

# Train Random Forest
rf_model <- randomForest(Species ~ ., data = iris, 
                         ntree = 500, 
                         importance = TRUE)

# View results
print(rf_model)

# Feature importance
importance(rf_model)
varImpPlot(rf_model)

# Out-of-Bag (OOB) Error Rate
# (Internal cross-validation estimate of error)
plot(rf_model, main = "OOB Error Rate vs Number of Trees")

Support Vector Machines (SVM)

Concept: SVM finds the optimal hyperplane that best separates classes in a high-dimensional space. It maximizes the margin (distance) between the hyperplane and the nearest data points (support vectors).

History: Developed by Vladimir Vapnik and colleagues at AT&T Bell Labs in 1992-1995, based on statistical learning theory.

Assumptions & Characteristics
  • Effective in high-dimensional spaces.
  • Uses the kernel trick to handle non-linear boundaries (RBF, Polynomial, Linear kernels).
  • Memory efficient (uses subset of training points).
  • Sensitive to feature scaling — must standardize data.
  • Can be used for both classification (SVC) and regression (SVR).
Applications
  • Handwriting recognition.
  • Face detection.
  • Text categorization.
R Implementation (Classification with iris)
# install.packages("e1071")
library(e1071)

data(iris)
set.seed(42)

# Train SVM with Radial Basis Function kernel
svm_model <- svm(Species ~ ., data = iris, 
                 kernel = "radial", 
                 cost = 1, 
                 gamma = 0.1)

# Summary of the model
summary(svm_model)

# Predictions
predictions <- predict(svm_model, iris)
table(Predicted = predictions, Actual = iris$Species)

# Tune hyperparameters
tuned <- tune(svm, Species ~ ., data = iris,
              ranges = list(cost = c(0.1, 1, 10, 100),
                            gamma = c(0.01, 0.1, 1)))
print(tuned)

K-Nearest Neighbors (KNN)

Concept: A non-parametric, lazy learning algorithm. To classify a new point, it finds the K closest training examples in feature space and assigns the most common class (classification) or the average value (regression).

History: First described by Fix and Hodges in 1951 as a "discriminatory analysis" technique for pattern recognition.

Assumptions & Characteristics
  • No training phase — computation happens at prediction time.
  • Highly sensitive to the choice of K and distance metric (Euclidean, Manhattan).
  • Must scale features — otherwise, variables with larger ranges dominate.
  • Computationally expensive for large datasets.
  • Curse of dimensionality — performance drops in high dimensions.
Applications
  • Recommendation systems.
  • Anomaly detection.
  • Missing value imputation.
R Implementation (Classification with iris)
# install.packages("class")
library(class)

data(iris)
set.seed(42)

# Split data
train_idx <- sample(1:nrow(iris), 0.7 * nrow(iris))
train_X <- iris[train_idx, -5]  # Features only
train_y <- iris[train_idx, 5]   # Labels
test_X  <- iris[-train_idx, -5]
test_y  <- iris[-train_idx, 5]

# IMPORTANT: Scale features
train_X_scaled <- scale(train_X)
test_X_scaled  <- scale(test_X, 
                        center = attr(train_X_scaled, "scaled:center"),
                        scale  = attr(train_X_scaled, "scaled:scale"))

# Run KNN with K = 5
knn_pred <- knn(train = train_X_scaled, 
                test  = test_X_scaled, 
                cl    = train_y, 
                k     = 5)

# Evaluate
table(Predicted = knn_pred, Actual = test_y)
mean(knn_pred == test_y)  # Accuracy

Supervised Regression in R

We will predict car fuel efficiency (mpg) using the built-in mtcars dataset with Random Forest Regression and evaluate using standard metrics.

# install.packages("randomForest")
# install.packages("Metrics")
library(randomForest)
library(Metrics)

data(mtcars)
set.seed(42)

# Split data (70% train, 30% test)
train_idx <- sample(1:nrow(mtcars), 0.7 * nrow(mtcars))
train_data <- mtcars[train_idx, ]
test_data  <- mtcars[-train_idx, ]

# Train Random Forest Regression model
rf_reg <- randomForest(mpg ~ ., data = train_data, ntree = 500)

# Make predictions on test set
predictions <- predict(rf_reg, test_data)

# Evaluation Metrics
cat("RMSE:", rmse(test_data$mpg, predictions), "\n")
cat("MAE :", mae(test_data$mpg, predictions), "\n")
cat("R²  :", 1 - sum((test_data$mpg - predictions)^2) / 
                  sum((test_data$mpg - mean(test_data$mpg))^2), "\n")

# Visualize: Actual vs Predicted
plot(test_data$mpg, predictions, 
     xlab = "Actual MPG", ylab = "Predicted MPG",
     main = "Random Forest Regression: Actual vs Predicted")
abline(0, 1, col = "red", lwd = 2) # Perfect prediction line
Key Regression Metrics Explained:
  • RMSE (Root Mean Squared Error): Average magnitude of errors. Penalizes large errors more.
  • MAE (Mean Absolute Error): Average absolute difference. More robust to outliers.
  • R² (Coefficient of Determination): Proportion of variance explained. Closer to 1 is better.

2. Unsupervised Learning

Unsupervised learning deals with unlabeled data. The algorithm explores the data to find hidden structures, patterns, or groupings without any prior knowledge of what the output should be.

Clustering Algorithms

K-Means Clustering

Concept: Partitions data into K distinct clusters based on distance to the nearest cluster mean (centroid). Each data point belongs to the cluster with the nearest mean.

History: The term "K-Means" was first used by James MacQueen in 1967, though the idea dates back to Hugo Steinhaus in 1957.

Algorithm Steps
  1. Randomly initialize K centroids.
  2. Assign each data point to the nearest centroid.
  3. Recalculate centroids as the mean of assigned points.
  4. Repeat steps 2-3 until centroids stop moving (convergence).
Assumptions & Characteristics
  • Assumes clusters are spherical and equally sized.
  • Sensitive to outliers and initial centroid placement.
  • Must choose K — use the Elbow Method or Silhouette Score.
  • Requires feature scaling.
Applications
  • Customer segmentation in marketing.
  • Document clustering.
  • Anomaly detection.
R Implementation (Using USArrests)
data(USArrests)

# IMPORTANT: Scale the data
scaled_data <- scale(USArrests)

# 1. Find optimal K using Elbow Method
wcss <- numeric(10)
for (i in 1:10) {
  km <- kmeans(scaled_data, centers = i, nstart = 25)
  wcss[i] <- km$tot.withinss
}
plot(1:10, wcss, type = "b", pch = 19,
     xlab = "Number of Clusters (K)",
     ylab = "Within-Cluster Sum of Squares",
     main = "Elbow Method for Optimal K")

# 2. Run K-Means with K = 4
set.seed(42)
km_model <- kmeans(scaled_data, centers = 4, nstart = 25)

# View cluster assignments and centers
print(km_model)

# Visualize clusters (using first 2 PCA dimensions)
library(factoextra)
fviz_cluster(km_model, data = scaled_data,
             main = "K-Means Clustering of US Arrests")

Hierarchical Clustering

Concept: Builds a hierarchy of clusters either through an agglomerative (bottom-up) or divisive (top-down) approach. The result is visualized as a dendrogram.

Agglomerative Approach (Most Common)
  1. Start with each point as its own cluster.
  2. Find the two closest clusters and merge them.
  3. Repeat until all points are in one cluster.
Linkage Methods
  • Single: Minimum distance between points.
  • Complete: Maximum distance between points.
  • Average (UPGMA): Average distance between all pairs.
  • Ward's: Minimizes variance within merged clusters.
Applications
  • Phylogenetic trees in biology.
  • Social network analysis.
  • Market segmentation.
R Implementation (Using USArrests)
data(USArrests)
scaled_data <- scale(USArrests)

# Compute distance matrix (Euclidean)
dist_matrix <- dist(scaled_data, method = "euclidean")

# Build hierarchical model using Ward's method
hc_model <- hclust(dist_matrix, method = "ward.D2")

# Plot dendrogram
plot(hc_model, 
     main = "Hierarchical Clustering of US Arrests",
     xlab = "US States", 
     ylab = "Height (Distance)",
     cex = 0.6, 
     hang = -1)

# Cut tree to get 4 clusters
rect.hclust(hc_model, k = 4, border = 2:5)

# Get cluster assignments
clusters <- cutree(hc_model, k = 4)
table(clusters)

Principal Component Analysis (PCA)

Concept: A dimensionality reduction technique that transforms correlated variables into a smaller set of uncorrelated variables called principal components, while retaining as much variance as possible.

History: Introduced by Harold Hotelling in the 1930s.

Why Use PCA?
  • Reduce noise and redundancy in data.
  • Visualize high-dimensional data in 2D/3D.
  • Speed up machine learning algorithms.
  • Handle multicollinearity.
Assumptions
  • Linear relationships between variables.
  • Features should be scaled (mean = 0, variance = 1).
  • Large variance = important information.
Applications
  • Image compression.
  • Genomics (gene expression analysis).
  • Finance (yield curve analysis).
R Implementation (Using USArrests)
data(USArrests)
scaled_data <- scale(USArrests)

# Perform PCA
pca_result <- prcomp(scaled_data, center = TRUE, scale. = TRUE)

# View summary (proportion of variance explained)
summary(pca_result)

# Scree plot
plot(pca_result, main = "Scree Plot")

# Biplot (visualize data in PC1 vs PC2 space)
biplot(pca_result, scale = 0, cex = 0.7,
       main = "PCA Biplot of US Arrests")

# Extract principal component scores
pc_scores <- pca_result$x
head(pc_scores)

# Visualize with factoextra
library(factoextra)
fviz_pca_ind(pca_result, 
             geom.ind = "point",
             col.ind = "cos2", 
             gradient.cols = c("#00AFBB", "#E7B800", "#FC4E07"),
             title = "PCA - Individuals colored by Quality")

DBSCAN (Density-Based Spatial Clustering)

Concept: Groups together points that are closely packed (high-density regions) and marks points in low-density regions as outliers/noise.

History: Proposed by Ester, Kriegel, Sander, and Xu in 1996.

Key Parameters
  • eps (ε): Maximum distance between two points to be considered neighbors.
  • minPts: Minimum number of points required to form a dense region (cluster).
Advantages over K-Means
  • Does not require specifying K.
  • Can find clusters of arbitrary shape (not just spherical).
  • Identifies outliers/noise automatically.
Applications
  • Spatial data analysis (crime hotspots).
  • Image segmentation.
  • Anomaly detection in networks.
R Implementation (Using iris)
# install.packages("dbscan")
library(dbscan)

data(iris)
# Use only numeric features
iris_numeric <- iris[, -5]
scaled_iris <- scale(iris_numeric)

# Run DBSCAN
# eps = 0.8, minPts = 5 (tune based on k-NN distance plot)
dbscan_model <- dbscan(scaled_iris, eps = 0.8, minPts = 5)

# View results
print(dbscan_model)

# Cluster assignments (-1 indicates noise/outliers)
table(dbscan_model$cluster)

# Visualize (using first 2 features for 2D plot)
plot(scaled_iris[, 1:2], 
     col = dbscan_model$cluster + 1,
     pch = 19, 
     main = "DBSCAN Clustering of Iris Data",
     xlab = names(iris)[1], ylab = names(iris)[2])
legend("topleft", 
       legend = unique(dbscan_model$cluster),
       col = 1:(length(unique(dbscan_model$cluster))),
       pch = 19)

3. Semi-Supervised Learning

Semi-supervised learning sits between supervised and unsupervised learning. It uses a small amount of labeled data combined with a large amount of unlabeled data to improve learning accuracy.

Historical Context

The concept emerged in the late 1990s and early 2000s, driven by the realization that in many real-world scenarios (e.g., web page classification, medical imaging), obtaining labels is expensive or time-consuming, but unlabeled data is abundant.

Pioneering work by Blum & Mitchell (1998) on Co-Training and Zhu & Ghahramani (2002) on Label Propagation laid the foundation.

When to Use
  • Labeled data is scarce or expensive to obtain.
  • Large amounts of unlabeled data are available.
  • The data has an underlying structure that can be exploited.
Common Applications
  • Speech analysis (audio is abundant, transcriptions are costly).
  • Protein sequence classification.
  • Web content classification.

Key Techniques

1. Self-Training
  1. Train a model on the small labeled dataset.
  2. Use it to predict labels for unlabeled data.
  3. Add the most confident predictions to the labeled set.
  4. Retrain and repeat.
2. Label Propagation / Spreading

Constructs a graph where nodes are data points and edges represent similarity. Labels from labeled nodes "propagate" through the graph to unlabeled nodes based on their connectivity.

3. Co-Training

Uses two classifiers trained on different "views" (feature subsets) of the data. Each classifier labels unlabeled data for the other to learn from.

4. Generative Models

Assume data comes from a mixture of distributions. Uses Expectation-Maximization (EM) to estimate parameters using both labeled and unlabeled data.

R Implementation (Self-Training Example)

# install.packages("SSLR")
# install.packages("randomForest")
library(SSLR)
library(randomForest)

data(iris)
set.seed(42)

# Simulate semi-supervised scenario:
# Only 10% of data is labeled, 90% is unlabeled (NA)
labeled_idx <- sample(1:nrow(iris), 15)
iris_labels <- iris$Species
iris_labels[-labeled_idx] <- NA  # Mark as unlabeled

# Prepare data (features + labels with NAs)
X <- iris[, -5]
y <- iris_labels

# Build Self-Training model using Random Forest as base learner
self_trainer <- SelfTraining(
  base_learner = randomForest::randomForest,
  threshold = 0.85,  # Confidence threshold
  max_iter = 20
)

# Fit the semi-supervised model
model <- self_trainer$fit(X, y)

# Predict on all data
predictions <- model$predict(X)

# Evaluate on originally labeled data
table(Predicted = predictions[labeled_idx], 
      Actual = iris$Species[labeled_idx])

cat("Accuracy on labeled data:", 
    mean(predictions[labeled_idx] == iris$Species[labeled_idx]), "\n")
Caution: Semi-supervised learning can fail if the initial model's predictions are wrong — errors propagate and compound. Always validate with a holdout labeled set.

4. Reinforcement Learning (RL)

Reinforcement Learning is a type of machine learning where an agent learns to make decisions by performing actions in an environment to maximize cumulative rewards over time.

Historical Context

The roots of RL lie in psychology — Edward Thorndike's "Law of Effect" (1911) and B.F. Skinner's operant conditioning (1930s).

Richard Bellman formalized the mathematics in the 1950s with Dynamic Programming and the Bellman Equation.

Modern RL exploded with Mnih et al. (2013) — DeepMind's Deep Q-Network that learned to play Atari games at superhuman levels, and AlphaGo (2016) which defeated world champion Lee Sedol at Go.

Core Concepts
  • Agent: The learner/decision-maker.
  • Environment: What the agent interacts with.
  • State (S): Current situation of the agent.
  • Action (A): What the agent can do.
  • Reward (R): Feedback signal from the environment.
  • Policy (π): Strategy the agent uses to choose actions.

The Bellman Equation

The Bellman Equation is the foundation of most RL algorithms. It defines the value of a state recursively:

$$ V(s) = \max_a \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s') \right] $$
  • $V(s)$: Value of being in state $s$.
  • $R(s,a)$: Immediate reward for action $a$ in state $s$.
  • $\gamma$ (gamma): Discount factor (0 to 1) — how much we care about future rewards.
  • $P(s'|s,a)$: Probability of transitioning to state $s'$.

Key RL Algorithms

1. Q-Learning (Model-Free)

Learns the value of each action in each state (Q-value) without knowing the environment's dynamics. Uses the update rule:

$$ Q(s,a) \leftarrow Q(s,a) + \alpha [r + \gamma \max_{a'} Q(s',a') - Q(s,a)] $$
2. SARSA (On-Policy)

Similar to Q-Learning but updates based on the actual action taken (not the best possible). More conservative, safer for risky environments.

3. Policy Gradient Methods

Directly optimize the policy (strategy) rather than value functions. Examples: REINFORCE, PPO, A3C.

4. Deep Q-Networks (DQN)

Combines Q-Learning with deep neural networks to handle high-dimensional state spaces (like pixels in video games).

Applications of Reinforcement Learning

Game AI
AlphaGo, Dota 2, Chess engines
Robotics
Autonomous navigation, manipulation
Finance
Algorithmic trading, portfolio optimization
Autonomous Vehicles
Self-driving cars, drones
Healthcare
Treatment optimization, drug dosing
Recommendation Systems
YouTube, Netflix, ads

R Implementation (Markov Decision Process)

We'll model a simple Markov Decision Process (MDP) — the mathematical framework for RL — using the markovchain package. Imagine a weather model where states are "Sunny", "Rainy", "Cloudy" and we want to find the optimal policy.

# install.packages("markovchain")
library(markovchain)

# Define states
states <- c("Sunny", "Rainy", "Cloudy")

# Define transition probability matrix
# Each row = current state, each column = next state
transition_matrix <- new("matrix", 
    c(0.6, 0.2, 0.2,   # From Sunny: 60% Sunny, 20% Rainy, 20% Cloudy
      0.3, 0.4, 0.3,   # From Rainy
      0.4, 0.3, 0.3),  # From Cloudy
    nrow = 3, ncol = 3, byrow = TRUE,
    dimnames = list(states, states))

# Create markovchain object
weather_mc <- new("markovchain", 
                  states = states,
                  transitionMatrix = transition_matrix,
                  name = "Weather")

# Display the transition matrix
print(weather_mc)

# Predict weather 3 days from now, starting from Sunny
initial_state <- c(1, 0, 0)  # 100% Sunny
predict(weather_mc, initial_state, 3)

# Steady-state probabilities (long-run distribution)
steadyStates(weather_mc)

# --------------------------------------------------------
# For full RL (Q-Learning), use the 'ReinforcementLearning' package
# --------------------------------------------------------
# install.packages("ReinforcementLearning")
library(ReinforcementLearning)

# Define a simple gridworld environment
# (Agent moves in a 3x3 grid to reach a goal)
control <- list(
  lr = 0.1,          # Learning rate (alpha)
  gamma = 0.9,       # Discount factor
  epsilon = 0.1      # Exploration rate
)

# Note: Full RL implementation requires defining:
# 1. State space
# 2. Action space
# 3. Reward function
# 4. Environment dynamics
# This is typically done with custom environment functions.
# See: https://cran.r-project.org/package=ReinforcementLearning
Note for Students: While R has RL capabilities, the industry standard for advanced RL (especially Deep RL) is Python with libraries like Stable-Baselines3, OpenAI Gym, and PyTorch. R is excellent for MDP modeling, Q-learning on small problems, and reinforcement learning in statistical contexts.
