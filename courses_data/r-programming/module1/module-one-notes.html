What is R?

R is a free, open-source programming language and software environment for statistical computing and graphics.

Created by Ross Ihaka and Robert Gentleman at the University of Auckland in 1993, R has evolved into the gold standard for statistical analysis. Unlike general-purpose languages like Python, R was built by statisticians, for statisticians.

Why Choose R?
  • Statistical Power: State-of-the-art packages for regression, time-series, and survival analysis.
  • Visualization: ggplot2 allows for publication-quality graphics.
  • Reproducibility: R Markdown integrates code, output, and text into dynamic reports.
Key Concepts
  • Open Source (CRAN)
  • Vectorized Operations
  • Extensive Package Ecosystem
  • Strong Community Support

Installation Guide

To start programming in R, you need two components: the R Engine (the brain) and RStudio (the interface).

Step 1: Install R (The Engine)

R is the underlying computational engine. You must install this first.

  1. Visit the Comprehensive R Archive Network (CRAN):
    https://cran.r-project.org/
  2. Select your operating system (Windows, MacOS, or Linux).
  3. Download the base installer and run it with default settings.
Step 2: Install RStudio (The IDE)

RStudio is an Integrated Development Environment (IDE) that makes working with R easier.

  1. Visit Posit (formerly RStudio):
    https://posit.co/download/
  2. Download the free "RStudio Desktop" version.
  3. Install and launch. It will automatically detect your R installation.

Core Data Structures

Understanding how R stores data is critical. Structures are defined by their dimension (1D, 2D, nD) and homogeneity (same type vs mixed types).

Vectors: The Foundation of R

A vector is a sequence of data elements of the same basic type. R is a vectorized language, meaning operations apply to entire vectors at once without loops.

Creation & Indexing
# Create a numeric vector
ages <- c(25, 30, 35, 40)

# Create a character vector
names <- c("Alice", "Bob", "Charlie")

# R uses 1-based indexing (not 0!)
print(ages[1])   # Output: 25
print(ages[2:4]) # Output: 30 35 40

# Negative indexing excludes elements
print(ages[-1])  # Output: 30 35 40
Vectorized Operations
v1 <- c(1, 2, 3)
v2 <- c(4, 5, 6)

# Element-wise addition
v1 + v2  # Output: 5 7 9

# Scalar multiplication
v1 * 10  # Output: 10 20 30

# Logical comparison
v1 > 2   # Output: FALSE FALSE TRUE
Note on Coercion: If you mix types (e.g., numbers and strings), R converts everything to the most flexible type (usually Character).
c(1, "Hello") becomes "1" "Hello".

Matrices: Two-Dimensional Data

A matrix is a collection of data elements arranged in a two-dimensional rectangular layout. All elements must be of the same type.

# Create a 3x3 matrix filled by row
mat <- matrix(
  data = 1:9, 
  nrow = 3, 
  ncol = 3,
  byrow = TRUE
)

# Accessing Elements [row, col]
mat[1, 2]   # Row 1, Col 2
mat[2, ]    # Entire Row 2
mat[, 3]    # Entire Column 3

Arrays: Multi-Dimensional Data

An array is a multi-dimensional generalization of a matrix. It can have 3 or more dimensions.

# Create a 3D array (2 rows, 3 cols, 2 matrices)
arr <- array(1:12, dim=c(2, 3, 2))

# Accessing elements
arr[1, 2, 1] # Row 1, Col 2 of the 1st matrix

Lists: Heterogeneous Collections

A list can contain elements of different types and even other lists. This is the most flexible structure in R.

my_list <- list(
  name = "John",
  age = 30,
  scores = c(85, 90, 92),
  is_student = TRUE
)

# Accessing elements
my_list$name       # Access by name
my_list[[1]]       # Access by index (returns content)
my_list[1]         # Returns a sub-list

Linear Algebra in R

R is exceptionally powerful for linear algebra, which underpins machine learning and statistical modeling.

Matrix Operations
A <- matrix(c(1, 2, 3, 4), nrow=2, ncol=2)
B <- matrix(c(5, 6, 7, 8), nrow=2, ncol=2)

# Element-wise Multiplication (Hadamard)
A * B

# True Matrix Multiplication (Dot Product)
A %*% B

# Transpose
t(A)
$$ A \times B \neq A \cdot B $$ Use %*% for linear algebra multiplication, not *.
Determinant & Inverse

Essential for solving systems of linear equations ($Ax=b$).

M <- matrix(c(4, 3, 3, 2), nrow=2, ncol=2)

# Calculate Determinant
det(M)  
# Output: -1 (Non-zero means invertible)

# Calculate Inverse
solve(M)

# Verify: M * Inverse(M) = Identity Matrix
M %*% solve(M)
Eigenvalues & Eigenvectors

Crucial for Principal Component Analysis (PCA).

C <- matrix(c(2, 1, 1, 2), nrow=2, ncol=2)
eigen_result <- eigen(C)

eigen_result$values  # Eigenvalues
eigen_result$vectors # Eigenvectors

Data Frames: The Workhorse of Data Science

A Data Frame is a list of vectors of equal length. It is the R equivalent of an Excel sheet or SQL table. Columns can have different types (numeric, character, factor).

Creating and Manipulating Data Frames
# Create a Data Frame
df <- data.frame(
  Name = c("Alice", "Bob", "Charlie"),
  Age = c(25, 30, 35),
  Salary = c(50000, 60000, 70000)
)

# View structure
str(df)

# Accessing Columns
df$Age
df[["Age"]]

# Filtering: Rows where Age > 25
subset_df <- df[df$Age > 25, ]

# Adding a new column
df$Bonus <- df$Salary * 0.10
Handling Missing Data (NA)
# Introduce missing value
df$Salary[2] <- NA

# Check for missing values
is.na(df$Salary)

# Fill missing values with Mean
mean_val <- mean(df$Salary, na.rm = TRUE)
df$Salary[is.na(df$Salary)] <- mean_val
Pro Tip: In professional settings, use the dplyr package for cleaner syntax: df %>% filter(Age > 25) %>% select(Name, Salary)

Data Importation and Cleaning

Before analysis, data must be efficiently imported into R and cleaned to ensure accuracy, consistency, and reliability.

Importing Data into R

R provides multiple robust ways to bring external data into your environment.

# Base R: Reading standard CSV files
data_csv <- read.csv("data/sales_report.csv")

# The 'readr' package (part of tidyverse): Much faster and cleaner
library(readr)
data_csv <- read_csv("data/sales_report.csv")

# The 'readxl' package: Reading Excel files without Java dependencies
library(readxl)
data_excel <- read_excel("data/sales_report.xlsx", sheet = "Q1")

# Viewing the first few rows
head(data_csv)
View(data_csv) # Opens an interactive spreadsheet viewer in RStudio
Initial Data Inspection

Always inspect your data immediately after importing to understand its structure and identify obvious issues.

# Check the structure (variable types, missing values)
str(data_csv)

# Statistical summary of all variables
summary(data_csv)

# Check dimensions (number of rows, number of columns)
dim(data_csv)

# Check and standardize column names
colnames(data_csv)
colnames(data_csv) <- tolower(colnames(data_csv)) # Make all lowercase
Data Cleaning Techniques

Real-world data is messy. Cleaning involves handling missing values, correcting data types, and standardizing formats.

# 1. Handling Missing Values (NA)
# Count total missing values in the dataset
sum(is.na(data_csv))

# Remove rows with ANY missing value
clean_data <- na.omit(data_csv)

# Or replace NAs with a specific value (e.g., mean for numeric columns)
data_csv$Revenue[is.na(data_csv$Revenue)] <- mean(data_csv$Revenue, na.rm = TRUE)

# 2. Changing Data Types
# Convert character to Date format
data_csv$Date <- as.Date(data_csv$Date, format = "%Y-%m-%d")

# Convert numeric/categorical text to Factor
data_csv$Region <- as.factor(data_csv$Region)

# 3. Removing Duplicates
clean_data <- unique(data_csv)
Best Practice: Never overwrite your original raw data file. Always assign cleaned data to a new object (e.g., clean_data) to maintain a reproducible data pipeline.

Data Manipulation using dplyr

dplyr is the cornerstone of the tidyverse. It provides a consistent, intuitive "grammar" of data manipulation with highly readable verbs.

The Pipe Operator (%>% or |>): The pipe takes the output of one function and passes it as the first argument to the next.
Example: x %>% f(y) is the same as f(x, y). It allows you to chain operations sequentially, making code read like a step-by-step recipe.
The Core dplyr Verbs
1. select(): Pick variables by name
# Select specific columns to keep
library(dplyr)
selected_data <- data_csv %>%
  select(Transaction_ID, Date, Revenue, Region)
2. filter(): Pick rows by condition
# Filter for high-value transactions in the North region
filtered_data <- data_csv %>%
  filter(Revenue > 1000, Region == "North")
3. mutate(): Add or modify columns
# Create a new column for Profit (assuming 20% margin)
mutated_data <- data_csv %>%
  mutate(Profit = Revenue * 0.20,
         Revenue_Category = ifelse(Revenue > 5000, "High", "Low"))
4. arrange(): Reorder rows
# Sort data by Revenue in descending order
arranged_data <- data_csv %>%
  arrange(desc(Revenue))
Aggregation: group_by() and summarise()

This is where dplyr truly shines. We can split the data into groups, apply a summary function, and combine the results back together.

# Calculate total revenue and average transaction value per Region
summary_stats <- data_csv %>%
  group_by(Region) %>%
  summarise(
    Total_Revenue = sum(Revenue, na.rm = TRUE),
    Avg_Transaction = mean(Revenue, na.rm = TRUE),
    Num_Transactions = n(),
    .groups = 'drop' # Drops grouping structure after summarising
  ) %>%
  arrange(desc(Total_Revenue)) # Sort the final summary table

print(summary_stats)
Pro Tip: You can chain all these verbs together for a complete, readable data transformation pipeline:
data %>% filter(!is.na(Revenue)) %>% group_by(Region) %>% summarise(Avg = mean(Revenue))

Data Visualization in R

Visualization is crucial for exploratory data analysis and communicating insights. R offers both fast base graphics and the incredibly powerful ggplot2 package.

Quick Exploratory Plots with Base R

Base R graphics are fast, require no additional packages, and are perfect for quick checks during your initial analysis phase.

# 1. Scatter Plot
plot(data_csv$Revenue, data_csv$Profit, 
     main = "Revenue vs. Profit",
     xlab = "Revenue ($)", ylab = "Profit ($)",
     col = "blue", pch = 16) # pch changes point shape

# 2. Histogram (Distribution of a continuous variable)
hist(data_csv$Revenue, 
     breaks = 20, 
     col = "lightblue",
     main = "Distribution of Revenue",
     xlab = "Revenue")

# 3. Boxplot (Comparing distributions across categorical groups)
boxplot(Revenue ~ Region, data = data_csv,
        col = "lightgreen",
        main = "Revenue by Region",
        ylab = "Revenue ($)")

# 4. Barplot (Categorical counts)
barplot(table(data_csv$Region),
        col = "salmon",
        main = "Transaction Count by Region")
Publication-Quality Graphics with ggplot2

ggplot2 builds plots layer by layer using the "Grammar of Graphics". It is the industry standard for data visualization in R.

The Anatomy of a ggplot
library(ggplot2)

# Basic structure: ggplot(data, aes(x, y)) + geom_...()
# aes() defines how variables are mapped to visual properties (aesthetics)
Common Geometries (Geoms)
# 1. Scatter Plot
ggplot(data_csv, aes(x = Revenue, y = Profit)) +
  geom_point(color = "darkblue", alpha = 0.6) + # alpha adds transparency
  labs(title = "Revenue vs. Profit", x = "Total Revenue", y = "Net Profit") +
  theme_minimal()

# 2. Boxplot with Fill Aesthetic
ggplot(data_csv, aes(x = Region, y = Revenue, fill = Region)) +
  geom_boxplot() +
  labs(title = "Revenue Distribution by Region") +
  theme_light() +
  theme(legend.position = "none") # Hides the legend

# 3. Bar Chart (Summarizing categorical data)
ggplot(data_csv, aes(x = Region)) +
  geom_bar(fill = "coral") +
  labs(title = "Number of Transactions per Region", x = "Region", y = "Count")

# 4. Line Plot (Time Series)
ggplot(daily_sales, aes(x = Date, y = Revenue)) +
  geom_line(color = "darkgreen", size = 1) +
  geom_point(size = 2) +
  theme_classic()
Advanced Customization: Faceting and Themes
# Faceting: Creating small multiples (subplots) based on a categorical variable
ggplot(data_csv, aes(x = Revenue, y = Profit)) +
  geom_point() +
  facet_wrap(~ Region) + # Creates a separate plot panel for each Region
  theme_bw() +           # Apply a clean Black & White theme
  labs(caption = "Source: Internal Sales Database 2026")
Remember: Every layer in ggplot2 is added with a + sign. Ensure the + is at the end of the line, not the beginning of the next line!

Control Flow & Functions

Conditional Statements

score <- 85

if (score >= 90) {
  print("Grade: A")
} else if (score >= 80) {
  print("Grade: B")
} else {
  print("Grade: C")
}

Loops (Use Sparingly!)

R is vectorized, so loops are often slower than vector operations. Use them only when necessary.

# For Loop
for (i in 1:5) {
  print(i)
}

# While Loop
count <- 1
while (count <= 5) {
  print(count)
  count <- count + 1
}
Writing Functions
calculate_area <- function(radius) {
  area <- pi * radius^2
  return(area)
}

# Call the function
result <- calculate_area(5)
print(result) # 78.53982
