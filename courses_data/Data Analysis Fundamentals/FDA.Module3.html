3.1 Introduction to Statistical Analysis & Hypothesis Testing

Statistical analysis allows us to move beyond descriptive summaries and make inferences about populations, test theories, and quantify uncertainty.

Hypothesis Testing Fundamentals

Hypothesis testing is a formal procedure for investigating our ideas about the world using data. It involves setting up two competing statements:

  • Null Hypothesis (H0): The default assumption (e.g., "The new drug has no effect", "There is no correlation between Age and Blood Pressure").
  • Alternative Hypothesis (HA): The claim you are trying to find evidence for (e.g., "The new drug lowers blood pressure").

We calculate a p-value. If the p-value is less than our chosen significance level (α, typically 0.05), we reject the null hypothesis, concluding the result is statistically significant.

Confidence Intervals (CI)

Instead of a single point estimate (like a sample mean), a Confidence Interval provides a range of plausible values for the true population parameter.

CI = Sample Statistic ± (Critical Value × Standard Error)

Interpretation: A 95% CI means that if we were to take 100 different samples and compute a 95% CI for each, approximately 95 of those intervals would contain the true population parameter.

Correlation Analysis

Correlation measures the strength and direction of a linear relationship between two continuous variables. It ranges from -1 to +1.

  • Pearson (r): Measures linear correlation. Assumes normality and is sensitive to outliers.
  • Spearman (ρ): Measures monotonic correlation based on ranks. Robust to outliers and non-normal data.

3.2 Linear Regression Modeling

Regression analysis is used to model the relationship between a dependent (response) variable and one or more independent (predictor) variables.

Simple Linear Regression

Models the relationship between one predictor (X) and one continuous response (Y).

Y = β0 + β1X + ε
  • β0: The y-intercept (value of Y when X = 0).
  • β1: The slope (change in Y for a 1-unit increase in X).
  • ε: The error term (residuals).
Multiple Linear Regression

Extends simple regression to include multiple predictors. It allows us to understand the effect of one variable while holding others constant.

Y = β0 + β1X1 + β2X2 + ... + βnXn + ε
Model Fit: R-Squared (R2)

R2 represents the proportion of the variance in the dependent variable that is predictable from the independent variables. An R2 of 0.80 means the model explains 80% of the variability of the response data around its mean.

R Implementation
library(tidyverse)

# Simple Linear Regression: Predicting MaxHR from Age
model_simple <- lm(MaxHR ~ Age, data = heart_data)
summary(model_simple)

# Multiple Linear Regression: Predicting MaxHR from Age, Cholesterol, and Sex
model_multiple <- lm(MaxHR ~ Age + Chol + Sex, data = heart_data)
summary(model_multiple)

# Extracting Coefficients and Confidence Intervals
confint(model_multiple, level = 0.95)

3.3 Logistic Regression & Classification Concepts

When the target variable is categorical (e.g., Yes/No, Disease/No Disease), we use classification algorithms. Logistic Regression is the foundational algorithm for binary classification.

Logistic Regression

Instead of predicting a continuous value, Logistic Regression predicts the probability that an observation belongs to a specific class. It uses the logit (sigmoid) function to squash the output between 0 and 1.

P(Y=1) = 1 / (1 + e-(β0 + β1X1 + ...))

Odds Ratios (OR): In R, exponentiating the logistic regression coefficients (exp(coef)) gives the Odds Ratio. An OR > 1 indicates that as the predictor increases, the odds of the event occurring increase.

Classification Metrics
MetricFormula / ConceptWhen to Prioritize
Accuracy Correct Predictions / Total Predictions When classes are perfectly balanced.
Precision True Positives / (True Positives + False Positives) When False Positives are costly (e.g., Spam detection).
Recall (Sensitivity) True Positives / (True Positives + False Negatives) When False Negatives are costly (e.g., Cancer screening).
F1-Score Harmonic mean of Precision and Recall When you need a balance between Precision and Recall.
R Implementation (using caret)
library(caret)

# Prepare data
heart_data$HD <- factor(heart_data$HD, levels = c(0, 1), labels = c("No", "Yes"))
set.seed(123)
trainIndex <- createDataPartition(heart_data$HD, p = 0.8, list = FALSE)
train_data <- heart_data[trainIndex, ]
test_data  <- heart_data[-trainIndex, ]

# Train Logistic Regression Model
set.seed(123)
model_glm <- train(HD ~ Age + MaxHR + Ca + Thal, 
                   data = train_data, 
                   method = "glm", 
                   family = "binomial",
                   trControl = trainControl(method = "cv", number = 10))

# Predictions and Confusion Matrix
predictions <- predict(model_glm, newdata = test_data)
confusionMatrix(predictions, reference = test_data$HD, positive = "Yes")

3.4 Clustering Concepts (Unsupervised Learning)

Unlike regression and classification, clustering is an unsupervised learning technique. There is no target variable; the goal is to discover hidden structures or natural groupings within the data.

K-Means Clustering

The most common clustering algorithm. It partitions n observations into k distinct, non-overlapping clusters based on feature similarity (usually Euclidean distance).

  1. Specify the number of clusters (k).
  2. Randomly initialize k centroids.
  3. Assign each data point to the nearest centroid.
  4. Recalculate the centroids as the mean of the assigned points.
  5. Repeat steps 3 and 4 until the centroids stop moving (convergence).
Determining the Optimal k: The Elbow Method

We plot the Within-Cluster Sum of Squares (WCSS) against different values of k. The "elbow" or inflection point of the curve indicates the optimal number of clusters where adding more clusters doesn't significantly improve the model.

R Implementation
library(factoextra)

# Select numeric features for clustering and scale them
cluster_data <- heart_data %>%
  select(Age, RestBP, Chol, MaxHR, Oldpeak) %>%
  na.omit() %>%
  scale()

# Compute K-Means with k=3
set.seed(123)
kmeans_result <- kmeans(cluster_data, centers = 3, nstart = 25)

# Visualize the clusters
fviz_cluster(kmeans_result, data = cluster_data,
             palette = c("#00AFBB", "#E7B800", "#FC4E07"),
             ggtheme = theme_minimal(),
             main = "K-Means Clustering of Patient Vitals")

3.5 Model Evaluation, Assumptions, and Interpretation

Building a model is only half the battle. Rigorous evaluation and assumption checking ensure your model is reliable, generalizable, and legally/ethically defensible.

Model Assumptions

  • Linearity: The relationship between X and Y is linear.
  • Independence: Observations are independent of each other.
  • Normality: The residuals (errors) are normally distributed.
  • Equal Variance (Homoscedasticity): The variance of residuals is constant across all levels of X.

  • Binary outcome variable.
  • Independence of observations.
  • Linearity of independent variables and the log-odds (not necessarily the probability).
  • Lack of severe multicollinearity among predictors.
Advanced Evaluation Techniques
  • Cross-Validation: Instead of a single train/test split, K-Fold Cross-Validation trains the model on k-1 folds and tests on the remaining fold, rotating k times. This provides a much more robust estimate of model performance.
  • ROC Curve & AUC: The Receiver Operating Characteristic curve plots the True Positive Rate (Recall) against the False Positive Rate (1 - Specificity) across all possible probability thresholds. The Area Under the Curve (AUC) summarizes this; an AUC of 0.5 is random guessing, while 1.0 is perfect.
  • Regression Metrics: For continuous outcomes, we use RMSE (Root Mean Squared Error) or MAE (Mean Absolute Error) to measure the average distance between predictions and actual values.
Plotting the ROC Curve in R
library(pROC)

# Get predicted probabilities for the positive class ("Yes")
probabilities <- predict(model_glm, newdata = test_data, type = "prob")[, "Yes"]

# Calculate and plot ROC
roc_obj <- roc(test_data$HD, probabilities)
plot(roc_obj, col = "#0d6efd", lwd = 2, main = "ROC Curve for Heart Disease Prediction")
auc(roc_obj) # Print the AUC value

3.6 Practical Session: End-to-End Predictive Modeling

Let's integrate statistical testing, regression, classification, and clustering into a comprehensive R workflow using the Heart.csv dataset.

Scenario: Comprehensive Clinical Data Analysis

We will explore correlations, predict continuous vitals using linear regression, classify disease presence using logistic regression, and segment patients using K-Means.

# ==============================================================================
# 1. SETUP AND DATA PREPARATION
# ==============================================================================
library(tidyverse)
library(caret)
library(pROC)
library(factoextra)
library(broom)

# Load data
heart <- read.csv("Heart.csv") %>% na.omit()
heart$HD <- factor(heart$HD, levels = c(0, 1), labels = c("No", "Yes"))

# ==============================================================================
# 2. CORRELATION ANALYSIS
# ==============================================================================
# Calculate correlation matrix for continuous variables
num_vars <- heart %>% select(Age, RestBP, Chol, MaxHR, Oldpeak)
cor_matrix <- cor(num_vars)

# Visualize using corrplot (if installed) or base R pairs plot
pairs(num_vars, col = heart$HD, pch = 16, 
      main = "Scatterplot Matrix of Continuous Variables by Disease Status")

# ==============================================================================
# 3. LINEAR REGRESSION: Predicting Maximum Heart Rate
# ==============================================================================
# Model: MaxHR is expected to decrease with Age and ST Depression (Oldpeak)
lm_model <- lm(MaxHR ~ Age + Oldpeak + Sex, data = heart)
summary(lm_model)

# Interpretation:
# If Age coefficient is -0.8, it means for every 1-year increase in age, 
# MaxHR decreases by 0.8 bpm, holding Oldpeak and Sex constant.

# ==============================================================================
# 4. LOGISTIC REGRESSION: Predicting Heart Disease (HD)
# ==============================================================================
set.seed(123)
trainIndex <- createDataPartition(heart$HD, p = 0.8, list = FALSE)
train <- heart[trainIndex, ]
test  <- heart[-trainIndex, ]

# Fit model using caret for built-in cross-validation
set.seed(123)
glm_model <- train(HD ~ Age + MaxHR + Ca + Thal + ChestPain, 
                   data = train, 
                   method = "glm", 
                   family = "binomial",
                   trControl = trainControl(method = "repeatedcv", number = 10, repeats = 3),
                   metric = "ROC")

# Evaluate on Test Set
glm_preds <- predict(glm_model, newdata = test)
cm <- confusionMatrix(glm_preds, reference = test$HD, positive = "Yes")
print(cm)

# Calculate Odds Ratios for interpretability
odds_ratios <- tidy(glm_model$finalModel) %>%
  filter(term != "(Intercept)") %>%
  mutate(OR = exp(estimate),
         OR_lower = exp(estimate - 1.96 * std.error),
         OR_upper = exp(estimate + 1.96 * std.error))
print(odds_ratios)

# Plot ROC Curve
glm_probs <- predict(glm_model, newdata = test, type = "prob")[, "Yes"]
roc_obj <- roc(test$HD, glm_probs)
plot(roc_obj, col = "#28a745", lwd = 2, main = "Logistic Regression ROC Curve")

# ==============================================================================
# 5. K-MEANS CLUSTERING: Patient Segmentation
# ==============================================================================
# Prepare data for clustering (must be scaled)
cluster_vars <- heart %>% select(Age, Chol, MaxHR, Oldpeak) %>% scale()

# Find optimal K using Elbow Method (WCSS)
wcss <- sapply(1:10, function(k) {
  kmeans(cluster_vars, centers = k, nstart = 25)$tot.withinss
})
plot(1:10, wcss, type = "b", pch = 19, frame = FALSE, 
     xlab = "Number of clusters (k)", ylab = "Within-Cluster Sum of Squares",
     main = "Elbow Method for Optimal k")

# Fit final K-Means (assuming k=3 based on elbow)
set.seed(123)
km_res <- kmeans(cluster_vars, centers = 3, nstart = 25)

# Visualize Clusters
fviz_cluster(km_res, data = cluster_vars,
             palette = c("#0d6efd", "#ffc107", "#dc3545"),
             ggtheme = theme_minimal(),
             main = "Patient Segmentation (K-Means)")

Exercise 3: Choosing the Right Evaluation Metric
Correct Answer:
  • a) The Accuracy Paradox: A naive model that simply predicts "Healthy" for every single patient will achieve 99% Accuracy. However, it is completely useless because it fails to identify the deadly disease. Accuracy is misleading in imbalanced datasets.
  • b) Prioritize Recall (Sensitivity): In medical screening, a False Negative (telling a sick patient they are healthy) can be fatal. Recall measures the proportion of actual sick patients that the model correctly identifies.
