3.1 Database Management

Before working with tables and data, you must first create the database environment that will house them.

CREATE DATABASE

Creates a new, empty database on the server.

CREATE DATABASE UniversityDB;
DROP DATABASE

Deletes an existing database along with all its tables and data. Use with extreme caution!

DROP DATABASE UniversityDB;
BACKUP DATABASE

Creates a full backup of an existing database for disaster recovery. Note: Syntax varies by RDBMS.

-- SQL Server
BACKUP DATABASE UniversityDB
TO DISK = 'D:\Backups\UniversityDB_Full.bak';

-- MySQL (Command Line)
-- mysqldump -u username -p UniversityDB > backup.sql

3.2 Table Management

CREATE TABLE

Defines a new table, its columns, and their respective data types.

CREATE TABLE Students (
    StudentID INT,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    EnrollmentDate DATE,
    GPA DECIMAL(3, 2)
);
ALTER TABLE

Modifies an existing table structure (adding, deleting, or changing columns).

-- Add a new column
ALTER TABLE Students ADD Email VARCHAR(100);

-- Modify a column's data type (Syntax varies by RDBMS)
ALTER TABLE Students MODIFY COLUMN GPA DECIMAL(4, 2); -- MySQL
ALTER TABLE Students ALTER COLUMN GPA DECIMAL(4, 2);  -- SQL Server

-- Drop a column
ALTER TABLE Students DROP COLUMN Email;
DROP TABLE

Deletes the table structure and all its data permanently.

DROP TABLE Students;

3.3 Constraints

Constraints are rules applied to table columns to ensure the accuracy, reliability, and integrity of the data.

ConstraintDescription
NOT NULLEnsures a column cannot have a NULL value.
UNIQUEEnsures all values in a column are different.
PRIMARY KEYA combination of NOT NULL and UNIQUE. Uniquely identifies each row.
FOREIGN KEYPrevents actions that would destroy links between tables.
CHECKEnsures that the values in a column satisfy a specific condition.
DEFAULTSets a default value for a column if none is specified.
Applying Constraints during Table Creation
CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50) NOT NULL,
    Email VARCHAR(100) UNIQUE,
    Age INT CHECK (Age >= 18),
    EnrollmentStatus VARCHAR(20) DEFAULT 'Active'
);
Foreign Key Constraint (Relational Integrity)
CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100) NOT NULL
);

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    -- Define Foreign Keys
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

3.4 Auto Increment

Auto-increment allows a unique number to be generated automatically when a new record is inserted into a table. This is often used for Primary Keys.

MySQL (AUTO_INCREMENT)
CREATE TABLE Students (
    StudentID INT AUTO_INCREMENT PRIMARY KEY,
    FirstName VARCHAR(50)
);
SQL Server (IDENTITY)
CREATE TABLE Students (
    StudentID INT IDENTITY(1,1) PRIMARY KEY,
    FirstName VARCHAR(50)
);
Note for PostgreSQL: PostgreSQL uses the SERIAL pseudo-type (e.g., StudentID SERIAL PRIMARY KEY) to achieve the same auto-incrementing behavior.

3.5 Working with Dates

SQL provides a rich set of functions for querying, formatting, and manipulating date and time data.

Common Date Functions
Function (MySQL / SQL Server)Description
NOW() / GETDATE()Returns the current date and time.
CURDATE() / CAST(GETDATE() AS DATE)Returns the current date only.
DATE_ADD() / DATEADD()Adds a specified time interval to a date.
DATEDIFF()Returns the number of days between two dates.
Date Formatting and Calculations
-- MySQL Examples
SELECT NOW();                                  -- Current Date & Time
SELECT DATE_FORMAT(NOW(), '%M %d, %Y');        -- Format: 'July 03, 2026'
SELECT DATEDIFF('2026-12-31', CURDATE());      -- Days until end of year
SELECT DATE_ADD(CURDATE(), INTERVAL 30 DAY);   -- Date 30 days from now

-- SQL Server Examples
SELECT FORMAT(GETDATE(), 'MMMM dd, yyyy');     -- Format: 'July 03, 2026'
SELECT DATEDIFF(day, GETDATE(), '2026-12-31'); -- Days until end of year
SELECT DATEADD(day, 30, GETDATE());            -- Date 30 days from now

3.6 Indexes and Views

Indexes (Performance Optimization)

Indexes are special lookup tables that the database search engine can use to speed up data retrieval. Think of them like the index at the back of a textbook.

-- Create an index on a single column
CREATE INDEX idx_lastname 
ON Students(LastName);

-- Create a unique index
CREATE UNIQUE INDEX idx_email 
ON Students(Email);

-- Drop an index
DROP INDEX idx_lastname ON Students;
Trade-off: Indexes speed up SELECT queries but slow down INSERT and UPDATE operations because the index must be updated every time the data changes.
Views (Virtual Tables)

A View is a virtual table based on the result-set of an SQL statement. It contains rows and columns, but does not store data itself (unless materialized).

-- Create a View
CREATE VIEW ActiveStudents AS
SELECT StudentID, FirstName, LastName, Email
FROM Students
WHERE EnrollmentStatus = 'Active';

-- Query the View just like a table
SELECT * FROM ActiveStudents 
WHERE LastName LIKE 'S%';

-- Drop a View
DROP VIEW ActiveStudents;
Use Cases: Simplify complex queries, restrict access to sensitive data (security), and present data in a specific format.
