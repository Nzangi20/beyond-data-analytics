1. Unsupervised Learning Basics

Unlike Supervised Learning, where the algorithm learns from labeled data (input features and known target outputs), Unsupervised Learning deals with unlabeled data. The algorithm is not told what the "correct answer" is. Instead, its goal is to explore the data, find hidden structures, identify patterns, and summarize the information.

Major Unsupervised Tasks

  • Clustering: Grouping similar data points together into distinct clusters (e.g., customer segmentation, biological species grouping).
  • Dimensionality Reduction: Reducing the number of features in a dataset while preserving as much variance/information as possible (e.g., PCA, t-SNE).
  • Anomaly Detection: Identifying rare items or events that differ significantly from the majority of the data (e.g., fraud detection).

1.1 The Four Paradigms of Clustering

Clustering algorithms can be broadly categorized into four distinct families based on how they define and construct clusters:

ParadigmCore ConceptKey AlgorithmPros & Cons
Centroid-Based Partitions data into non-hierarchical groups based on distance to a central point (centroid). K-Means Pros: Fast, scalable.
Cons: Requires pre-specifying $K$, sensitive to outliers, assumes spherical clusters.
Connectivity-Based Builds a hierarchy of clusters based on distance or connectivity (tree structure). Agglomerative Hierarchical Pros: No need to pre-specify $K$, intuitive dendrograms.
Cons: Computationally expensive for large datasets.
Density-Based Defines clusters as dense regions separated by sparse regions. Handles noise naturally. DBSCAN Pros: Finds arbitrary shapes, identifies outliers.
Cons: Struggles with varying densities, requires careful parameter tuning.
Distribution-Based Assumes data is generated from a mixture of underlying probability distributions. Gaussian Mixture Models (GMM) Pros: Soft clustering (probabilities), mathematically rigorous.
Cons: Prone to overfitting, computationally intensive.

In this module, we will use the Palmer Penguins dataset. This dataset contains morphological measurements of three penguin species (Adelie, Gentoo, Chinstrap) observed on three islands in the Palmer Archipelago, Antarctica.

3. Data Preprocessing & Scaling

Clustering algorithms rely heavily on distance calculations (like Euclidean distance). If features are on different scales (e.g., body mass in grams vs. bill depth in millimeters), the feature with the larger numerical range will dominate the clustering process. Therefore, feature scaling is mandatory.

Step 1: Handling Missing Values & Feature Selection

Python
# Check missing values
print("Missing Values:\n", df.isnull().sum())

# Drop rows with missing values to ensure clean distance calculations
df_clean = df.dropna()

# Select numerical features for clustering
numerical_features = ['bill_length_mm', 'bill_depth_mm', 'flipper_length_mm', 'body_mass_g']
X = df_clean[numerical_features].values

# Extract true species labels for external validation later
y_true = df_clean['species'].values
print(f"Cleaned data shape: {X.shape}")

Step 2: Feature Standardization

We use StandardScaler to transform the data so that each feature has a mean of 0 and a standard deviation of 1.

Python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

print(f"Mean after scaling: {X_scaled.mean(axis=0).round(4)}")
print(f"Std after scaling: {X_scaled.std(axis=0).round(4)}")
Output

4. K-Means Clustering (Centroid-Based)

K-Means aims to partition $n$ observations into $k$ clusters in which each observation belongs to the cluster with the nearest mean (cluster centroid). The objective is to minimize the Within-Cluster Sum of Squares (WCSS or Inertia):

$$ WCSS = \sum_{i=1}^{k} \sum_{x \in C_i} ||x - \mu_i||^2 $$

Where $\mu_i$ is the centroid of cluster $C_i$.

Step 1: Finding the Optimal $K$ (Elbow & Silhouette)

Python
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, davies_bouldin_score

K_range = range(2, 11)
inertias = []
silhouette_scores = []

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X_scaled)
    inertias.append(kmeans.inertia_)
    silhouette_scores.append(silhouette_score(X_scaled, kmeans.labels_))

# Plotting the Elbow and Silhouette curves
fig, axes = plt.subplots(1, 2, figsize=(14, 5))
axes[0].plot(K_range, inertias, 'bo-')
axes[0].set_title('Elbow Method (Optimal K)')
axes[0].set_xlabel('Number of Clusters (k)')
axes[0].set_ylabel('Inertia (WCSS)')

axes[1].plot(K_range, silhouette_scores, 'go-')
axes[1].set_title('Silhouette Score')
axes[1].set_xlabel('Number of Clusters (k)')
axes[1].set_ylabel('Silhouette Score')
plt.tight_layout()
plt.show()

Interpretation: The Elbow plot typically shows a sharp drop at $k=2$ and $k=3$, with diminishing returns after $k=3$. The Silhouette score usually peaks at $k=2$ or $k=3$. Given our domain knowledge (3 penguin species), we select $K=3$.

Step 2: Fitting K-Means and Visualization

Python
# Fit K-Means with K=3
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans_labels = kmeans.fit_predict(X_scaled)

# Visualize using PCA (2D Projection)
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

plt.figure(figsize=(8, 6))
scatter = plt.scatter(X_pca[:, 0], X_pca[:, 1], c=kmeans_labels, cmap='viridis', s=60, alpha=0.8)
plt.colorbar(scatter, label='Cluster')
plt.title('K-Means Clusters (PCA-reduced)')
plt.xlabel('First Principal Component')
plt.ylabel('Second Principal Component')
plt.show()

5. Hierarchical Clustering (Connectivity-Based)

Hierarchical clustering builds a multilevel hierarchy of clusters by merging or splitting them successively. We use Agglomerative Clustering (bottom-up approach), where each point starts as its own cluster, and pairs of clusters are merged as one moves up the hierarchy.

🌳 The Dendrogram

A dendrogram is a tree diagram that visualizes the sequence of fusions and the distance at which they occurred. It allows us to "cut" the tree at a specific height to obtain a desired number of clusters without pre-specifying $K$.

Step 1: Plotting the Dendrogram

Python
from scipy.cluster.hierarchy import dendrogram, linkage

# Calculate the linkage matrix using Ward's method (minimizes variance within clusters)
linked = linkage(X_scaled, method='ward')

plt.figure(figsize=(12, 6))
dendrogram(linked, truncate_mode='lastp', p=30, leaf_rotation=90., leaf_font_size=10.)
plt.title('Hierarchical Clustering Dendrogram (Ward Linkage)')
plt.xlabel('Sample Index (or Cluster Size)')
plt.ylabel('Ward Distance')
# Draw a line to indicate cutting the tree at K=3
plt.axhline(y=15, color='r', linestyle='--', label='Cut for K=3')
plt.legend()
plt.show()

Step 2: Fitting Agglomerative Clustering

Python
from sklearn.cluster import AgglomerativeClustering

# Fit Hierarchical Clustering
agg_clust = AgglomerativeClustering(n_clusters=3, linkage='ward')
hierarchical_labels = agg_clust.fit_predict(X_scaled)

print(f"Hierarchical Cluster Counts:\n{pd.Series(hierarchical_labels).value_counts()}")

6. DBSCAN (Density-Based)

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) groups together points that are closely packed together (points with many nearby neighbors), marking as outliers points that lie alone in low-density regions.

Key Parameters

  • eps ($\epsilon$): The maximum distance between two samples for one to be considered as in the neighborhood of the other.
  • min_samples: The number of samples in a neighborhood for a point to be considered as a core point.
⚠️ DBSCAN & High Dimensions

DBSCAN struggles with the "Curse of Dimensionality" because distance metrics become less meaningful in high-dimensional spaces. It works best on 2D or 3D data, or data reduced via PCA. For our 4D penguin data, we will apply it directly but acknowledge its limitations compared to K-Means.

Python
from sklearn.cluster import DBSCAN
from sklearn.neighbors import NearestNeighbors

# 1. Find optimal eps using K-Distance graph
neighbors = NearestNeighbors(n_neighbors=5)
neighbors_fit = neighbors.fit(X_scaled)
distances, indices = neighbors_fit.kneighbors(X_scaled)
distances = np.sort(distances[:, 4], axis=0) # 5th nearest neighbor distance

plt.figure(figsize=(8, 5))
plt.plot(distances)
plt.title('K-Distance Graph (k=5)')
plt.xlabel('Data Points sorted by distance')
plt.ylabel('Epsilon (distance)')
plt.show()

# 2. Fit DBSCAN (Based on the elbow in the K-Distance graph, e.g., eps=1.2)
dbscan = DBSCAN(eps=1.2, min_samples=5)
dbscan_labels = dbscan.fit_predict(X_scaled)

# Check for noise points (labeled as -1)
n_noise_ = list(dbscan_labels).count(-1)
print(f"Number of noise points (outliers): {n_noise_}")
print(f"DBSCAN Cluster Counts:\n{pd.Series(dbscan_labels).value_counts()}")

7. Gaussian Mixture Models (Distribution-Based)

GMM assumes that all the data points are generated from a mixture of a finite number of Gaussian distributions with unknown parameters. Unlike K-Means, which assigns a point to exactly one cluster (Hard Clustering), GMM provides a probability of belonging to each cluster (Soft Clustering).

The probability that a point $x$ belongs to cluster $k$ is determined by the Expectation-Maximization (EM) algorithm:

$$ P(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x | \mu_k, \Sigma_k) $$

Where $\pi_k$ is the mixing weight, and $\mathcal{N}$ is the Gaussian distribution.

Step 1: Selecting Components using BIC/AIC

Python
from sklearn.mixture import GaussianMixture

n_components_range = range(1, 7)
bics = []
aics = []

for n in n_components_range:
    gmm = GaussianMixture(n_components=n, random_state=42)
    gmm.fit(X_scaled)
    bics.append(gmm.bic(X_scaled))
    aics.append(gmm.aic(X_scaled))

plt.figure(figsize=(8, 5))
plt.plot(n_components_range, bics, 'bo-', label='BIC')
plt.plot(n_components_range, aics, 'go-', label='AIC')
plt.title('GMM: BIC and AIC for Model Selection')
plt.xlabel('Number of Components')
plt.ylabel('Information Criterion')
plt.legend()
plt.show()

Interpretation: The lowest BIC/AIC score indicates the optimal model complexity. We typically select the $K$ where the curve bottoms out (usually $K=3$ for this dataset).

Step 2: Fitting GMM and Soft Clustering

Python
# Fit GMM with 3 components
gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=42)
gmm_labels = gmm.fit_predict(X_scaled)

# Get probabilities (Soft Clustering)
probabilities = gmm.predict_proba(X_scaled)
print("Probabilities for the first 5 samples:")
print(pd.DataFrame(probabilities, columns=['Prob_Cluster_0', 'Prob_Cluster_1', 'Prob_Cluster_2']).head())

8. Dimensionality Reduction: Principal Component Analysis (PCA)

While clustering groups data points based on similarity, Dimensionality Reduction aims to reduce the number of features in a dataset while preserving as much variance (information) as possible. This is crucial for visualizing high-dimensional data, reducing computational cost, and mitigating the "Curse of Dimensionality".

Principal Component Analysis (PCA) is the most widely used linear dimensionality reduction technique. It transforms the original correlated variables into a new set of uncorrelated variables called Principal Components, which are ordered by the amount of variance they explain.

Key Concepts

  • Principal Components: New orthogonal axes that capture the maximum variance in the data. PC1 captures the most variance, PC2 captures the second most, and so on.
  • Explained Variance Ratio: A metric that tells us exactly how much of the dataset's total information (variance) is captured by each individual component.
  • Standardization: PCA is highly sensitive to the scale of the features. It is mandatory to scale data (e.g., using StandardScaler) before applying PCA, otherwise, features with larger numerical ranges will dominate the principal components.

Applying PCA to the Penguin Dataset

Let's reduce our 4-dimensional penguin measurements down to 2 dimensions so we can easily visualize the natural groupings (which we know correspond to the 3 species) on a 2D scatter plot.

Python
from sklearn.decomposition import PCA

# Initialize PCA to retain 2 principal components
pca = PCA(n_components=2)

# Fit and transform the scaled data
X_pca = pca.fit_transform(X_scaled)

# Create a DataFrame for easier visualization
pca_df = pd.DataFrame(data=X_pca, columns=['PC1', 'PC2'])
pca_df['True_Species'] = y_true

# Check how much variance is explained by each component
print("Explained Variance Ratio:", pca.explained_variance_ratio_)
print("Total Variance Explained by 2 PCs:", sum(pca.explained_variance_ratio_))

# Visualize the 2D projection
plt.figure(figsize=(10, 6))
sns.scatterplot(data=pca_df, x='PC1', y='PC2', hue='True_Species', palette='husl', s=100, alpha=0.8)
plt.title('PCA: 2D Projection of Penguin Species', fontsize=14, fontweight='bold')
plt.xlabel(f'Principal Component 1 ({pca.explained_variance_ratio_[0]:.2%} Variance)')
plt.ylabel(f'Principal Component 2 ({pca.explained_variance_ratio_[1]:.2%} Variance)')
plt.legend(title='Species')
plt.grid(True, alpha=0.3)
plt.show()
📊 Interpreting the PCA Plot

Even without using the species labels during the PCA transformation, the 2D scatter plot reveals three distinct, well-separated clusters! This confirms that the morphological differences between Adelie, Gentoo, and Chinstrap penguins are strong enough to be captured by just the first two principal components. Notice how Gentoo penguins are clearly separated on the right (high PC1 values, driven by larger flipper length and body mass), while Adelie and Chinstrap are separated along PC2 (driven by bill dimensions).

9. Comparative Evaluation

To objectively compare the four clustering algorithms, we use both Internal Metrics (based on the data structure) and External Metrics (comparing against the true species labels).

Metrics Definitions

  • Silhouette Score (Internal): Measures how similar an object is to its own cluster compared to other clusters. Range: [-1, 1]. Higher is better.
  • Davies-Bouldin Index (Internal): Measures the average similarity between each cluster and its most similar one. Lower is better.
  • Adjusted Rand Index (ARI) (External): Measures the similarity between two data clusterings, adjusted for chance. Range: [-1, 1]. Higher is better.
  • Normalized Mutual Information (NMI) (External): Measures the amount of information shared between the predicted clusters and true labels. Range: [0, 1]. Higher is better.
Python
from sklearn.metrics import (silhouette_score, davies_bouldin_score, 
                             calinski_harabasz_score, adjusted_rand_score, 
                             normalized_mutual_info_score)

# Store results
results = {
    'K-Means': kmeans_labels,
    'Hierarchical': hierarchical_labels,
    'DBSCAN': dbscan_labels,
    'GMM': gmm_labels
}

evaluation_data = []

for name, labels in results.items():
    # Handle DBSCAN noise points (-1) for internal metrics if necessary, 
    # but scikit-learn handles it by ignoring noise or treating it as a cluster.
    if name == 'DBSCAN' and len(set(labels)) <= 1:
        continue # Skip if DBSCAN failed to find more than 1 cluster
        
    sil = silhouette_score(X_scaled, labels)
    dbi = davies_bouldin_score(X_scaled, labels)
    ch = calinski_harabasz_score(X_scaled, labels)
    ari = adjusted_rand_score(y_true, labels)
    nmi = normalized_mutual_info_score(y_true, labels)
    
    evaluation_data.append({
        'Algorithm': name,
        'Silhouette': round(sil, 3),
        'Davies-Bouldin': round(dbi, 3),
        'Calinski-Harabasz': round(ch, 1),
        'ARI': round(ari, 3),
        'NMI': round(nmi, 3)
    })

eval_df = pd.DataFrame(evaluation_data)
print(eval_df)

🏆 Key Takeaways from Evaluation

K-Means and Hierarchical (Ward) perform almost identically because Ward's linkage mathematically minimizes the exact same WCSS objective that K-Means optimizes. GMM edges them out slightly because it accounts for the elliptical shape of the clusters (covariance) rather than assuming perfect spheres. DBSCAN underperforms here because penguin species form distinct, compact blobs rather than arbitrary density-connected shapes, and 4 dimensions dilute the concept of "density".

When to use which algorithm?

  • Use K-Means when you need speed, have large datasets, and expect spherical clusters of similar size.
  • Use Hierarchical when you need to understand the data hierarchy, have a small-to-medium dataset, and want to visualize relationships via dendrograms.
  • Use DBSCAN when you suspect your data has arbitrary shapes (e.g., rings, spirals) or when you need to automatically filter out noise/anomalies.
  • Use GMM when you need "soft" probabilities of cluster membership, or when clusters overlap and have different elliptical shapes.
  • Use PCA when you need to visualize high-dimensional data, reduce computational complexity, or remove multicollinearity before feeding data into supervised models.
