1.1 Introduction to SQL

SQL (Structured Query Language) is the standard language for accessing and manipulating databases. It allows you to create, read, update, and delete data in relational database management systems (RDBMS).

Database Concepts
  • Database: A structured collection of data.
  • Table: A collection of related data held in a structured format (rows and columns).
  • Row (Record): A single entry in a table.
  • Column (Field): A specific piece of data within a table.
SQL Syntax Rules
-- This is a single-line comment
/* This is a 
   multi-line comment */

SELECT FirstName, LastName
FROM Employees
WHERE Department = 'IT';
Key Rules: SQL keywords are NOT case-sensitive, but it is convention to write them in UPPERCASE. Statements end with a semicolon (;). String values must be enclosed in single quotes.

1.2 Retrieving Data

SELECT & SELECT DISTINCT

The SELECT statement specifies which columns to retrieve. Use * for all columns (though specifying columns is best practice).

-- Select specific columns
SELECT FirstName, LastName, Department
FROM Employees;

-- Select unique values only
SELECT DISTINCT Department
FROM Employees;
WHERE & ORDER BY

Use WHERE to filter records, and ORDER BY to sort them.

-- Filter and Sort
SELECT FirstName, Salary
FROM Employees
WHERE Department = 'IT'
ORDER BY Salary DESC;
Note: DESC sorts in descending order. Omit it or use ASC for ascending (default).

1.3 Logical Operators

Logical operators combine multiple conditions in a WHERE clause.

AND

Both conditions must be TRUE.

WHERE Department = 'IT' 
AND Salary > 75000;
OR

At least one condition must be TRUE.

WHERE Department = 'IT' 
OR Department = 'HR';
NOT

Reverses the result of a condition.

WHERE NOT Department = 'IT';
-- or
WHERE Department <> 'IT';
Pro Tip: Always use parentheses () when combining AND and OR to control the order of evaluation.

1.4 Filtering Data

LIKE & Wildcards

Search for patterns using % (multiple chars) or _ (single char).

-- Starts with 'J'
WHERE FirstName LIKE 'J%'

-- Second character is 'a'
WHERE FirstName LIKE '_a%'
IN & BETWEEN

Filter by lists or ranges.

-- In a list
WHERE Department IN ('IT', 'HR')

-- In a range (inclusive)
WHERE Salary BETWEEN 60000 AND 80000
NULL Values
-- Find missing values
WHERE City IS NULL

-- Find non-missing values
WHERE City IS NOT NULL
Warning: Never use = NULL. Always use IS NULL.

1.5 Limiting Results

Different database systems use different syntax to limit rows:

-- SQL Server / MS Access
SELECT TOP 5 FirstName, Salary FROM Employees ORDER BY Salary DESC;

-- MySQL, PostgreSQL, SQLite
SELECT FirstName, Salary FROM Employees ORDER BY Salary DESC LIMIT 5;

-- Oracle
SELECT FirstName, Salary FROM Employees WHERE ROWNUM <= 5 ORDER BY Salary DESC;
Pagination: Use LIMIT 5 OFFSET 5 to skip the first 5 rows and return the next 5.

1.6 Aliases

Aliases give a table or column a temporary name for readability.

-- Column Alias
SELECT FirstName AS "First Name", 
       Salary * 12 AS "Annual Salary"
FROM Employees;

-- Table Alias
SELECT e.FirstName, e.Department 
FROM Employees AS e
WHERE e.Salary > 70000;
