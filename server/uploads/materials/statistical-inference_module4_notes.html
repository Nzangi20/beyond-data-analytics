Part A: Nonparametric Inference

Parametric tests (like the t-test and ANOVA) assume that your data follows a specific distribution, usually the Normal distribution. But what happens when your data is heavily skewed, contains extreme outliers, or is measured on an ordinal scale (like a Likert scale)? Nonparametric methods provide the solution.

Why and When to Use Nonparametric Tests?

Nonparametric tests are often called distribution-free tests. They do not assume a specific underlying distribution for the data.

When to use them:
  • The data is severely skewed or has heavy tails.
  • The sample size is very small, making it impossible to verify the normality assumption.
  • The data is ordinal (e.g., pain scales, customer satisfaction ratings) rather than continuous.
  • There are extreme outliers that cannot be justified for removal and heavily distort the mean.
Trade-off: If your data is perfectly normal, parametric tests (like the t-test) are mathematically more powerful (more likely to detect a true effect) than their nonparametric counterparts. Nonparametric tests achieve robustness by sacrificing a small amount of statistical power.
1. The Sign Test (One-Sample & Paired)

The simplest nonparametric test. It ignores the magnitude of the differences and only looks at the direction (the sign: positive or negative).

Clinical Example:

We measure patient pain scores (0-10 ordinal scale) before and after a new physical therapy. We want to know if the therapy generally reduces pain.

How it works:
  1. Calculate the difference for each pair: Di = Post - Pre.
  2. Count the number of positive differences (pain got worse) and negative differences (pain got better).
  3. Ignore any zeros (no change).
  4. Under the null hypothesis (no effect), we expect a 50/50 split. We use the Binomial distribution to test if the observed proportion of negative signs is significantly greater than 0.5.
# R: Paired Sign Test
pre_pain <- c(8, 7, 9, 6, 8, 7, 9, 8, 7, 6)
post_pain <- c(7, 6, 8, 6, 7, 5, 8, 7, 6, 5)

# Using the BSDA package
library(BSDA)
SIGN.test(pre_pain, post_pain, alternative = "greater")
2. Wilcoxon Signed-Rank Test (Paired / One-Sample)

The Wilcoxon Signed-Rank test is the nonparametric equivalent of the Paired t-test. Unlike the Sign Test, it considers both the sign and the magnitude of the differences by ranking them.

How it works:
  1. Calculate the differences Di.
  2. Rank the absolute values of the differences from smallest to largest.
  3. Attach the original sign (+ or -) to each rank.
  4. Sum the positive ranks (W+) and the negative ranks (W-).
  5. The test statistic W is the smaller of the two sums. If the treatment has a real effect, one sum will be drastically larger than the other.
# R: Wilcoxon Signed-Rank Test
# Testing if median difference is zero
wilcox.test(pre_pain, post_pain, paired = TRUE, alternative = "greater")
# Python: Wilcoxon Signed-Rank Test
from scipy.stats import wilcoxon
stat, p_value = wilcoxon(pre_pain, post_pain, alternative='greater')
print(f"Statistic: {stat}, P-value: {p_value}")
3. Mann-Whitney U Test (Independent Two-Sample)

The nonparametric equivalent of the Independent Two-Sample t-test. It tests whether two independent groups come from the same distribution (specifically, if one group tends to have larger values than the other).

Clinical Example:

Comparing the number of days it takes to recover from a surgery using Technique A vs. Technique B. The recovery days are heavily right-skewed (most recover in 3 days, but a few take 30+ days).

How it works:
  1. Pool all observations from both groups together.
  2. Rank the entire pooled dataset from 1 to N.
  3. Sum the ranks for Group A (RA) and Group B (RB).
  4. Calculate the U statistic, which represents the number of times a value from Group A precedes a value from Group B.
# R: Mann-Whitney U Test (wilcox.test in R)
group_A <- c(3, 4, 3, 5, 4, 30, 3) # Skewed with an outlier
group_B <- c(5, 6, 5, 7, 6, 8, 7)

wilcox.test(group_A, group_B, alternative = "less")
# Python: Mann-Whitney U Test
from scipy.stats import mannwhitneyu
stat, p_value = mannwhitneyu(group_A, group_B, alternative='less')
print(f"U-Statistic: {stat}, P-value: {p_value}")
4. Kruskal-Wallis Test (Independent >2 Samples)

The nonparametric equivalent of One-Way ANOVA. It tests if there are statistically significant differences between the medians of three or more independent groups.

How it works:
  1. Pool all data from all k groups and rank them from 1 to N.
  2. Calculate the sum of ranks for each group.
  3. The H statistic measures the variance of these rank sums. If all groups are identical, the rank sums will be roughly equal.
  4. The H statistic follows a Chi-Square distribution with k-1 degrees of freedom.
# R: Kruskal-Wallis Test
therapy_A <- c(4, 5, 4, 6, 5)
therapy_B <- c(6, 7, 6, 8, 7)
therapy_C <- c(3, 4, 3, 5, 4)

# Combine into a data frame
df_kw <- data.frame(
    score = c(therapy_A, therapy_B, therapy_C),
    group = rep(c("A", "B", "C"), each = 5)
)

kruskal.test(score ~ group, data = df_kw)
5. Spearman Rank Correlation

The nonparametric equivalent of Pearson Correlation. While Pearson measures linear relationships between continuous variables, Spearman measures monotonic relationships (as one variable increases, the other consistently increases or decreases, even if not in a straight line).

How it works:

It is literally just the Pearson correlation coefficient calculated on the ranks of the data rather than the raw values.

ρs = 1 - [ (6 × Σdi2) / (n(n2 - 1)) ]

Where di is the difference between the ranks of corresponding variables.

# R: Spearman Correlation
age <- c(20, 30, 40, 50, 60, 70)
pain_score <- c(2, 3, 4, 5, 7, 9) # Monotonic but not perfectly linear

cor.test(age, pain_score, method = "spearman")

Part B: Regression Inference

In Module 3, we learned how to fit a regression line. But fitting a line is just the beginning. Regression inference allows us to quantify the uncertainty of our model, test if the relationships are real, and predict future outcomes with calculated confidence.

1. Introduction to Linear Regression & OLS

The Simple Linear Regression model assumes that the relationship between a predictor (X) and a response (Y) can be described by a straight line, plus some random error.

Yi = β0 + β1Xi + εi
  • β0: The true population y-intercept.
  • β1: The true population slope (the change in Y for a 1-unit increase in X).
  • εi: The random error term, assumed to be normally distributed with mean 0 and constant variance σ2.

We estimate these unknown population parameters using Ordinary Least Squares (OLS), which finds the line that minimizes the sum of the squared vertical distances (residuals) between the observed data points and the line.

2. Inference for Regression Coefficients

Because our sample is just one of many possible samples, our estimated slope (β̂1) is a random variable. It has its own sampling distribution.

If the assumptions of linear regression hold, the sampling distribution of β̂1 is Normal:

β̂1 ~ N( β1 , σ2 / Σ(Xi - X̄)2 )

The standard error of the slope, SE(β̂1), measures how much the estimated slope would vary from sample to sample. It decreases (becomes more precise) if the data points are spread far apart along the X-axis, or if the residual variance (σ2) is small.

3. Confidence Intervals & Hypothesis Testing
Confidence Interval for the Slope

We use the t-distribution with n-2 degrees of freedom to construct a confidence interval for the true population slope:

β̂1 ± tα/2, n-2 × SE(β̂1)

Interpretation: "We are 95% confident that for every 1-unit increase in X, the true average change in Y is between [Lower Bound] and [Upper Bound]."

Hypothesis Testing (The t-test)

To test if there is a statistically significant linear relationship between X and Y:

  • H0: β1 = 0 (No linear relationship)
  • HA: β1 ≠ 0 (There is a linear relationship)

The test statistic is:

t = β̂1 / SE(β̂1)

If the p-value associated with this t-statistic is < 0.05, we reject the null hypothesis and conclude that the predictor is statistically significant.

4. Coefficient of Determination (R²)

R2 measures the proportion of the variance in the dependent variable (Y) that is predictable from the independent variable (X).

R2 = SSR / SST = 1 - (SSE / SST)
  • SST (Total Sum of Squares): Total variability in Y.
  • SSR (Regression Sum of Squares): Variability explained by the model.
  • SSE (Error Sum of Squares): Unexplained variability (residuals).

An R2 of 0.85 means that 85% of the variation in Y is explained by the linear relationship with X, while 15% remains unexplained (due to other factors or random noise).

5. Model Diagnostics (Checking the LINE Assumptions)

Regression inference is only valid if the underlying assumptions hold. We use the acronym LINE to remember them:

AssumptionMeaningHow to Check (Visual)
Linearity The relationship between X and Y is linear. Residuals vs. Fitted Plot: Should show a random scatter of points around the horizontal line at 0. A curved pattern indicates non-linearity.
Independence The observations (and their residuals) are independent of each other. Checked via study design. For time-series data, check a Residuals vs. Order Plot for autocorrelation.
Normality The residuals are normally distributed. Normal Q-Q Plot: Points should closely follow the diagonal dashed line. Deviations at the tails indicate skewness or heavy tails.
Equal Variance
(Homoscedasticity)		 The variance of the residuals is constant across all levels of X. Scale-Location Plot: The spread of points should be roughly equal across the x-axis. A funnel shape indicates heteroscedasticity. 
# R: Running Regression and Diagnostics
model_lm <- lm(MaxHR ~ Age, data = heart)

# View Inference Summary (Coefficients, p-values, R-squared)
summary(model_lm)

# Extract 95% Confidence Intervals for coefficients
confint(model_lm, level = 0.95)

# Generate the 4 Diagnostic Plots
par(mfrow = c(2, 2))
plot(model_lm)

Part C: Advanced Topics

Modern statistics is no longer just about looking up critical values in a textbook table. Computational power has given rise to resampling methods, Bayesian philosophy, and a strict emphasis on reproducibility.

1. Bootstrap Methods

The Bootstrap is a powerful resampling technique used to estimate the sampling distribution of any statistic (mean, median, odds ratio, etc.) by sampling with replacement from the original dataset.

The Bootstrap Algorithm:
  1. Draw a random sample of size n with replacement from your original dataset. (This is called a "bootstrap sample". Some original data points will appear multiple times, and some won't appear at all).
  2. Calculate the statistic of interest (e.g., the median) for this bootstrap sample.
  3. Repeat steps 1 and 2 B times (usually B = 1000 or 10000).
  4. The distribution of these B statistics approximates the true sampling distribution. You can calculate the standard error or construct a 95% Confidence Interval by taking the 2.5th and 97.5th percentiles.
Why is this useful? If you want a confidence interval for the median or the difference in medians, there is no simple mathematical formula. The Bootstrap provides a computational workaround that requires zero distributional assumptions!
2. Permutation Tests

Also known as randomization tests, these are used to build an exact null distribution without relying on theoretical distributions like the t-distribution or F-distribution.

How it works:
  1. Calculate your observed test statistic (e.g., the difference in means between Treatment and Control).
  2. Shuffle (permute) the group labels randomly. (e.g., randomly reassign the "Treatment" and "Control" tags to the patients).
  3. Recalculate the test statistic for this shuffled data.
  4. Repeat this thousands of times to build a distribution of what the test statistic looks like when the null hypothesis is definitely true (since shuffling destroys any real treatment effect).
  5. The p-value is simply the proportion of permuted statistics that are as extreme as or more extreme than your original observed statistic.
3. Introduction to Bayesian Inference

Frequentist statistics (everything we've done so far) treats parameters (like μ or β) as fixed, unknown constants. Bayesian statistics treats parameters as random variables that have their own probability distributions.

The Bayesian Philosophy:

You start with a Prior Distribution (your belief about the parameter before seeing the data). You then collect data, and use Bayes' Theorem to update your belief, resulting in the Posterior Distribution.

Posterior ∝ Likelihood × Prior

Clinical Interpretation: "Based on prior clinical trials (Prior), and the new data from our 500 patients (Likelihood), there is a 95% probability that the true efficacy of the drug lies between 60% and 75% (Posterior)." This is a much more intuitive statement for clinicians than a Frequentist 95% Confidence Interval!

4. Reporting Results & Reproducibility
Best Practices for Reporting:
  • Always report Effect Sizes: A p-value only tells you if an effect exists. The effect size (e.g., Cohen's d, Odds Ratio, β coefficient) tells you if it matters.
  • Prefer Confidence Intervals over p-values: CIs provide a range of plausible values and convey the precision of the estimate.
  • Avoid "p-hacking": Do not run 20 different statistical tests and only report the one that yielded p < 0.05. Pre-register your analysis plan.
Reproducible Statistical Analysis:

A study is reproducible if another researcher can take your raw data and your code, and arrive at the exact same conclusions. To achieve this:

  • Use Literate Programming tools like R Markdown / Quarto, or Jupyter Notebooks. These combine your code, output, and written narrative into a single document.
  • Always set a random seed (set.seed(42)) at the top of your script for any stochastic processes (train/test splits, bootstrapping, MCMC).
  • Use version control (Git/GitHub) to track changes to your analysis code over time.

Practical Session: Computational Inference

Let's combine nonparametric testing, regression inference, and bootstrapping into a comprehensive R and Python workflow.

R Implementation: Nonparametric & Regression
# ==============================================================================
# 1. NONPARAMETRIC TESTING
# ==============================================================================
# Scenario: Does MaxHR differ significantly between Chest Pain types?
# Since MaxHR might not be perfectly normal across all groups, we use Kruskal-Wallis.

kruskal_res <- kruskal.test(MaxHR ~ ChestPain, data = heart)
print(kruskal_res)

# Post-hoc pairwise comparisons using Wilcoxon with Bonferroni correction
pairwise.wilcox.test(heart$MaxHR, heart$ChestPain, p.adjust.method = "bonferroni")

# Spearman Correlation: Is there a monotonic relationship between Age and Oldpeak?
cor_res <- cor.test(heart$Age, heart$Oldpeak, method = "spearman")
cat("Spearman Rho:", cor_res$estimate, "\n")
cat("P-value:", cor_res$p.value, "\n")

# ==============================================================================
# 2. REGRESSION INFERENCE
# ==============================================================================
# Predicting MaxHR based on Age
model_lm <- lm(MaxHR ~ Age, data = heart)

# Full statistical summary
summary(model_lm)
# Interpretation: Look at the 'Estimate' for Age (the slope), its 'Std. Error',
# the 't value', and the 'Pr(>|t|)' p-value.

# ==============================================================================
# 3. BOOTSTRAPPING A CONFIDENCE INTERVAL FOR THE MEDIAN
# ==============================================================================
library(boot)

# Define a function to calculate the median
median_func <- function(data, indices) {
  d <- data[indices] # Sample with replacement
  return(median(d))
}

# Run the bootstrap (1000 replications)
set.seed(123)
boot_results <- boot(data = heart$MaxHR, statistic = median_func, R = 1000)

# View the Bootstrap Confidence Intervals
boot.ci(boot_results, type = c("norm", "basic", "perc", "bca"))
Python Implementation: Statsmodels & Scipy
import pandas as pd
import numpy as np
import scipy.stats as stats
import statsmodels.api as sm
import statsmodels.formula.api as smf

# Load Data
df = pd.read_csv("Heart.csv").dropna()

# ==============================================================================
# 1. NONPARAMETRIC TESTING
# ==============================================================================
# Mann-Whitney U: Compare MaxHR between Sex (0 vs 1)
group_0 = df[df['Sex'] == 0]['MaxHR']
group_1 = df[df['Sex'] == 1]['MaxHR']

stat, p_val = stats.mannwhitneyu(group_0, group_1, alternative='two-sided')
print(f"Mann-Whitney U P-value: {p_val:.4f}")

# Spearman Correlation
rho, p_val_spearman = stats.spearmanr(df['Age'], df['Oldpeak'])
print(f"Spearman Rho: {rho:.4f}, P-value: {p_val_spearman:.6f}")

# ==============================================================================
# 2. REGRESSION INFERENCE
# ==============================================================================
# Using Statsmodels Formula API (similar to R's lm)
model = smf.ols('MaxHR ~ Age', data=df).fit()

# Print the comprehensive regression table
print(model.summary())
# The summary includes the coef, std err, t, P>|t|, and the 95% Conf. Intervals

# ==============================================================================
# 3. BOOTSTRAPPING IN PYTHON
# ==============================================================================
def bootstrap_median(data, n_iterations=10000):
    np.random.seed(123)
    medians = []
    for _ in range(n_iterations):
        sample = np.random.choice(data, size=len(data), replace=True)
        medians.append(np.median(sample))
    return np.array(medians)

boot_medians = bootstrap_median(df['MaxHR'].values)

# Calculate 95% Percentile Confidence Interval
ci_lower = np.percentile(boot_medians, 2.5)
ci_upper = np.percentile(boot_medians, 97.5)

print(f"Bootstrap 95% CI for Median MaxHR: [{ci_lower:.2f}, {ci_upper:.2f}]")

Exercise 1: Choosing the Right Nonparametric Test
Correct Answer:

Wilcoxon Signed-Rank Test (Paired).

Reasoning: The data is paired (same students measured twice). Because the differences are heavily skewed (violating the normality assumption required for the Paired t-test), the nonparametric Wilcoxon Signed-Rank test is the appropriate choice. The Sign Test could also be used, but Wilcoxon is more powerful as it considers the magnitude of the ranks.

Exercise 2: Interpreting Regression Output
Correct Answer:
  • a) Equation: Stay Duration̂ = 2.5 + 0.15 × Age
  • b) Slope Interpretation: For every additional year of age, the predicted hospital stay duration increases by 0.15 days (or roughly 3.6 hours), on average. Because the p-value < 0.001, this relationship is highly statistically significant.
  • c) R-squared Interpretation: 42% of the variability in hospital stay duration can be explained by the patient's age. The remaining 58% is due to other factors not included in the model (e.g., severity of illness, complications).
Exercise 3: Regression Diagnostics
Correct Answer:
  • Violation 1: Linearity. The "U-shape" in the Residuals vs. Fitted plot indicates that the true relationship is curved (non-linear), not a straight line. Fix: Add a polynomial term (e.g., X2) to the model, or apply a transformation to X or Y (like log or square root).
  • Violation 2: Normality of Residuals. The S-curve in the Q-Q plot indicates heavy tails (outliers) in the residuals. Fix: Investigate and potentially remove influential outliers, or use a robust regression method / nonparametric alternative.
Exercise 4: Bootstrap Concept
Correct Answer:

There is no simple, closed-form mathematical formula to calculate the standard error of the IQR, making traditional parametric confidence intervals impossible to construct.

The Bootstrap Solution: You would draw 10,000 random samples of size 50 with replacement from your original dataset. For each of the 10,000 samples, you calculate the IQR. The 2.5th percentile and 97.5th percentile of those 10,000 IQR values form your empirical 95% Bootstrap Confidence Interval.
