1.1 Introduction to Data Analysis

Data analysis is the process of inspecting, cleansing, transforming, and modeling data with the goal of discovering useful information, informing conclusions, and supporting decision-making.

The Data Analysis Workflow

Modern data analysis follows a structured, iterative lifecycle. While frameworks vary, the core steps remain consistent:

  1. Ask: Define the problem, objectives, and key questions. What business or clinical decision will this analysis inform?
  2. Collect: Gather data from primary sources (surveys, experiments) or secondary sources (databases, APIs, public repositories).
  3. Process (Clean): Handle missing values, remove duplicates, fix errors, and standardize formats. (This module's primary focus).
  4. Analyze: Apply statistical methods, exploratory data analysis (EDA), or machine learning algorithms to find patterns.
  5. Share: Visualize findings and communicate insights to stakeholders through reports and dashboards.
  6. Act: Use the insights to make informed, data-driven decisions and implement changes.
Data Sources and Collection Methods
Source TypeDescriptionExamples

1.2 Types of Data and Levels of Measurement

Understanding the nature of your variables is the most critical step in data preparation. The data type dictates which cleaning techniques, visualizations, and statistical tests are mathematically valid.

1. Qualitative (Categorical) Data

Data that represents characteristics or categories. It cannot be measured on a numerical scale that supports meaningful arithmetic.

  • Nominal: Categories with no inherent order (e.g., Blood Type, Country, Marital Status).
  • Ordinal: Categories with a meaningful order or ranking, but unequal intervals (e.g., Pain Scale 1-10, Education Level, Likert Scales).
2. Quantitative (Numerical) Data

Data that represents counts or measurements. Arithmetic operations are meaningful.

  • Discrete: Countable, usually whole numbers. (e.g., Number of children, Number of hospital visits).
  • Continuous: Measurable, can take any value within a range, including decimals. (e.g., Height, Weight, Temperature, Blood Pressure).
Stevens' Levels of Measurement (NOIR)

Identity only. No order. Stats: Mode, Frequency. R Class: factor or character.

Identity + Order. Intervals are unknown. Stats: Median, Percentiles. R Class: ordered factor.

Identity + Order + Equal Intervals. No true zero. Stats: Mean, SD. R Class: numeric.

Identity + Order + Equal Intervals + Absolute Zero. Ratios are meaningful. Stats: All. R Class: numeric.

1.3 Data Import and Export in R

R provides powerful tools for reading and writing data. While Base R has built-in functions, the tidyverse ecosystem (specifically readr, readxl, and haven) offers significantly faster and more robust alternatives.

Importing Data

library(readxl)
library(haven)

# 1. CSV Files (Using readr - much faster than base read.csv)
df_csv <- read_csv("data/patient_records.csv")

# 2. Excel Files (Using readxl - no Java dependencies required)
df_excel <- read_excel("data/financial_report.xlsx", sheet = "Q1_Summary")

df_spss <- read_sav("data/survey_data.sav")
df_stata <- read_dta("data/economic_data.dta")

# 4. R Data Files (.rds or .RData)
df_rds <- read_rds("data/cleaned_model_data.rds")

# Export to Excel

1.4 Data Cleaning: Missing Values & Outliers

Real-world data is inherently messy. "Garbage in, garbage out" is the golden rule of data science. Cleaning is often the most time-consuming, yet most critical, phase of the analytical workflow.

1. Handling Missing Values

Missing data (represented as NA in R) is not just an annoyance; it can introduce severe bias if not handled correctly. We must first understand why data is missing:

TypeDefinitionClinical/Business ExampleR Strategy
MCAR
(Missing Completely at Random)		 The missingness has no relationship with any observed or unobserved data. A lab machine randomly fails to process a blood sample. Listwise deletion (drop_na()) is generally safe.
MAR
(Missing at Random)		 The missingness is related to observed data, but not the missing data itself. Older patients are less likely to report their income (missingness depends on Age, not Income). Imputation using predictive models (e.g., mice package).
MNAR
(Missing Not at Random)		 The missingness is related to the unobserved missing data itself. People with very high incomes refuse to report their income on a survey. Requires domain expertise. Create a "Missing" indicator category or use specialized models.
Handling Missing Data in R
library(tidyverse)

# Count missing values per column
df_csv %>% summarise(across(everything(), ~ sum(is.na(.))))

# Strategy 1: Drop rows with ANY missing value (Listwise Deletion)
df_clean <- df_csv %>% drop_na()

# Strategy 2: Drop rows where a SPECIFIC key column is missing
df_clean <- df_csv %>% drop_na(Target_Variable)

# Strategy 3: Replace missing values with a constant (e.g., 0 or "Unknown")
df_clean <- df_csv %>%
  mutate(
    Income = replace_na(Income, 0),
    Category = replace_na(Category, "Unknown")
  )

# Strategy 4: Impute with Mean/Median (for continuous variables)
df_clean <- df_csv %>%
  mutate(
    Age = ifelse(is.na(Age), median(Age, na.rm = TRUE), Age)
  )
2. Detecting and Treating Outliers

Outliers are extreme values that deviate significantly from other observations. They can be genuine anomalies (e.g., a rare disease) or data entry errors (e.g., typing 200 instead of 20 for Age).

Method A: The Interquartile Range (IQR) Rule

Robust to non-normal distributions. Uses the 25th (Q1) and 75th (Q3) percentiles.

IQR = Q3 - Q1
Lower Bound = Q1 - 1.5 × IQR
Upper Bound = Q3 + 1.5 × IQR
Method B: The Z-Score Method

Assumes the data is approximately normally distributed. Identifies points that are multiple standard deviations away from the mean.

Z = (X - μ) / σ
Outliers are typically defined as |Z| > 3
Treating Outliers in R
# Visualizing Outliers with a Boxplot
ggplot(df_csv, aes(y = Income)) + 
  geom_boxplot(fill = "lightblue", color = "black") +
  theme_minimal() +
  labs(title = "Boxplot of Income to Detect Outliers")

# Treating Outliers using the IQR Method (Winsorization / Capping)
Q1 <- quantile(df_csv$Income, 0.25, na.rm = TRUE)
Q3 <- quantile(df_csv$Income, 0.75, na.rm = TRUE)
IQR_val <- IQR(Q3, Q1)

lower_bound <- Q1 - 1.5 * IQR_val
upper_bound <- Q3 + 1.5 * IQR_val

# Cap the outliers at the bounds (Winsorization)
df_csv <- df_csv %>%
  mutate(
    Income_Capped = case_when(
      Income < lower_bound ~ lower_bound,
      Income > upper_bound ~ upper_bound,
      TRUE ~ Income
    )
  )

# Alternatively, remove outliers entirely
df_no_outliers <- df_csv %>%
  filter(Income >= lower_bound & Income <= upper_bound)

1.5 Data Transformation and Validation

Once data is clean, it must be transformed into the correct format and scale required for statistical modeling and machine learning algorithms.

1. Data Coding and Recoding

Often, raw data needs to be grouped, binned, or relabeled to make it analytically useful.

library(dplyr)

# Recoding categorical variables (e.g., cleaning messy survey responses)
df_clean <- df_csv %>%
  mutate(
    Education_Clean = recode(Education_Raw,
      "High School Grad" = "High School",
      "HS Diploma" = "High School",
      "Bachelors Degree" = "Bachelor's",
      "BS" = "Bachelor's"
    )
  )

# Creating Ordinal Factors (Crucial for statistical modeling in R)
df_clean <- df_clean %>%
  mutate(
    Pain_Level = factor(Pain_Score, 
                        levels = c(1, 2, 3, 4, 5), 
                        labels = c("None", "Mild", "Moderate", "Severe", "Extreme"),
                        ordered = TRUE)
  )

# Binning Continuous Variables (e.g., Age groups)
df_clean <- df_clean %>%
  mutate(
    Age_Group = cut(Age, 
                    breaks = c(0, 18, 35, 50, 65, 100), 
                    labels = c("Minor", "Young Adult", "Adult", "Senior", "Elderly"))
  )
2. Conditional Transformation with case_when()

The case_when() function is R's equivalent to SQL's CASE WHEN or Excel's nested IF. It is the most readable way to create complex categorical variables.

df_clean <- df_clean %>%
  mutate(
    Risk_Category = case_when(
      Age > 65 & Chol > 240 ~ "High Risk",
      Age > 50 & Chol > 200 ~ "Moderate Risk",
      Age <= 50 & Chol <= 200 ~ "Low Risk",
      is.na(Chol) ~ "Unknown Risk",
      TRUE ~ "Standard Risk" # Default fallback condition
    )
  )
3. Data Validation and Quality Assessment

Before moving to analysis, perform a final sanity check to ensure logical consistency.

library(janitor) # Excellent package for data cleaning and validation

# 1. Check for duplicate rows
df_clean <- df_clean %>% get_dupes(everything())
# If no duplicates, it returns an empty tibble.

# 2. Standardize column names (make them snake_case, lowercase)
df_clean <- df_clean %>% clean_names()

# 3. Check for logical inconsistencies
# E.g., Discharge Date cannot be before Admission Date
logical_errors <- df_clean %>%
  filter(Discharge_Date < Admission_Date)

# 4. Verify factor levels and counts
table(df_clean$Risk_Category, useNA = "ifany")

1.6 Practical Session: End-to-End Data Preparation

Let's combine everything we've learned into a comprehensive, reproducible R script using the tidyverse pipeline.

Scenario: Preparing a Messy Clinical Dataset

We have a dataset messy_clinical_data.csv with missing ages, extreme cholesterol outliers, and messy text entries. We need to prepare it for a predictive model.

# ==============================================================================
# 1. SETUP AND IMPORT
# ==============================================================================
library(tidyverse)
library(janitor)

# Import and immediately clean column names
raw_data <- read_csv("data/messy_clinical_data.csv") %>%
  clean_names()

# Initial inspection
glimpse(raw_data)

# ==============================================================================
# 2. DATA CLEANING (Missing Values & Outliers)
# ==============================================================================
clean_data <- raw_data %>%
  # Handle Missing Values
  mutate(
    # Impute missing Age with the median
    age = ifelse(is.na(age), median(age, na.rm = TRUE), age),
    
    # Replace missing categorical data with "Unknown"
    smoking_status = replace_na(smoking_status, "Unknown")
  ) %>%
  # Handle Outliers in Cholesterol (IQR Capping)
  mutate(
    chol_q1 = quantile(cholesterol, 0.25, na.rm = TRUE),
    chol_q3 = quantile(cholesterol, 0.75, na.rm = TRUE),
    chol_iqr = chol_q3 - chol_q1,
    chol_lower = chol_q1 - 1.5 * chol_iqr,
    chol_upper = chol_q3 + 1.5 * chol_iqr,
    
    # Cap the outliers
    cholesterol_capped = case_when(
      cholesterol < chol_lower ~ chol_lower,
      cholesterol > chol_upper ~ chol_upper,
      TRUE ~ cholesterol
    )
  ) %>%
  # Remove the temporary helper columns
  select(-starts_with("chol_"))

# ==============================================================================
# 3. DATA TRANSFORMATION & RECODING
# ==============================================================================
final_data <- clean_data %>%
  # Recode messy text entries
  mutate(
    gender = recode(gender, "M" = "Male", "F" = "Female", "m" = "Male", "f" = "Female"),
    
    # Create a new categorical variable based on clinical guidelines
    bp_category = case_when(
      systolic_bp < 120 & diastolic_bp < 80 ~ "Normal",
      systolic_bp >= 120 & systolic_bp < 130 ~ "Elevated",
      systolic_bp >= 130 | diastolic_bp >= 80 ~ "Hypertension",
      TRUE ~ "Unknown"
    ),
    
    # Convert bp_category to an ordered factor
    bp_category = factor(bp_category, 
                         levels = c("Normal", "Elevated", "Hypertension", "Unknown"),
                         ordered = TRUE)
  ) %>%
  # Drop rows where the target variable is missing
  drop_na(target_disease)

# ==============================================================================
# 4. VALIDATION AND EXPORT
# ==============================================================================
# Final sanity check
cat("Final Dataset Dimensions:", dim(final_data), "\n")
cat("Missing Values Remaining:", sum(is.na(final_data)), "\n")

# Export the clean dataset
write_csv(final_data, "outputs/final_clean_clinical_data.csv")
write_rds(final_data, "outputs/final_clean_clinical_data.rds")

cat("✅ Data preparation complete! Ready for modeling.\n")
