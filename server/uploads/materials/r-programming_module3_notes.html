Historical Context of Regression

Regression is not just a mathematical tool; it is a concept born from observing the natural world and the need to predict planetary orbits and human traits.

The Origins: Gauss & Legendre (1805)

The mathematical foundation of regression—the Method of Least Squares—was first published by Adrien-Marie Legendre in 1805 and independently by Carl Friedrich Gauss in 1809. They used it to predict the orbits of planets and comets from astronomical observations, minimizing the sum of squared errors between observed and predicted positions.

"Regression to Mediocrity": Francis Galton (1886)

The actual term "Regression" was coined by Sir Francis Galton in the 19th century. While studying the heights of parents and their children, he noticed that exceptionally tall parents tended to have children who were shorter than them (closer to the average), and vice versa. He called this phenomenon "regression to mediocrity" (now known as regression to the mean).

The Logistic Function: Pierre-François Verhulst (1838)

The mathematical S-curve used in logistic regression was first described by Belgian mathematician Pierre-François Verhulst between 1838 and 1847. He created the "logistic equation" to model population growth, recognizing that populations grow exponentially at first but eventually level off due to resource constraints (carrying capacity).

Generalized Linear Models: Nelder & Wedderburn (1972)

Logistic regression as a unified statistical framework was formalized in 1972 by John Nelder and Robert Wedderburn. They introduced the term Generalized Linear Models (GLMs), which elegantly combined linear regression, logistic regression, and Poisson regression under one mathematical umbrella using "link functions" and maximum likelihood estimation.

Linear Regression Analysis

Linear regression models the relationship between a continuous dependent variable ($Y$) and one or more independent variables ($X$) by fitting a linear equation to observed data.

1. Simple Linear Regression (SLR)

Concept & Equation

SLR models the relationship between one predictor and one continuous outcome.

$$ Y = \beta_0 + \beta_1X + \epsilon $$
  • $Y$: Dependent variable (Response)
  • $X$: Independent variable (Predictor)
  • $\beta_0$: Y-intercept (Value of $Y$ when $X=0$)
  • $\beta_1$: Slope (Change in $Y$ for a 1-unit increase in $X$)
  • $\epsilon$: Error term (Residuals)
Core Assumptions (LINE)
  • Linearity: The relationship between $X$ and $Y$ is linear.
  • Independence: Observations are independent of each other.
  • Normality: The residuals ($\epsilon$) are normally distributed.
  • Equal Variance (Homoscedasticity): The variance of residuals is constant across all levels of $X$.
R Implementation (Using `cars` dataset)

The built-in cars dataset contains the speed of cars and the distances taken to stop.

# Load built-in data
data(cars)

# Fit Simple Linear Regression: Predict distance based on speed
model_slr <- lm(dist ~ speed, data = cars)

# View comprehensive summary
summary(model_slr)

# Interpretation:
# Coefficients: Estimate for 'speed' tells us how much 
# 'dist' increases for every 1 mph increase in speed.
# Multiple R-squared: Tells us the % of variance in 'dist' 
# explained by 'speed'.

# Predict stopping distance for a car going 25 mph
predict(model_slr, newdata = data.frame(speed = 25))

2. Multiple Linear Regression (MLR)

Concept & Equation

MLR extends SLR by including two or more predictors. It allows us to understand the effect of each predictor while holding the others constant.

$$ Y = \beta_0 + \beta_1X_1 + \beta_2X_2 + \dots + \beta_pX_p + \epsilon $$
Additional Assumptions
  • No Multicollinearity: Predictors should not be highly correlated with each other. If they are, it becomes impossible to isolate their individual effects.
  • No Autocorrelation: Residuals should not be correlated with each other (crucial for time-series data).
R Implementation (Using `mtcars` dataset)

The built-in mtcars dataset contains fuel consumption (`mpg`) and 10 other aspects of automobile design. We will predict `mpg` using weight (`wt`), horsepower (`hp`), and cylinders (`cyl`).

# Load built-in data
data(mtcars)

# Fit Multiple Linear Regression
model_mlr <- lm(mpg ~ wt + hp + cyl, data = mtcars)

# View summary
summary(model_mlr)

# Interpretation:
# Look at the p-values for each predictor. 
# If 'hp' has a p-value > 0.05, it means horsepower 
# is not a statistically significant predictor of mpg 
# when weight and cylinders are already in the model.

# Check overall model significance via F-statistic at the bottom.

3. Diagnostic Tests for Linear Regression

A model is only as good as its assumptions. We must visually and statistically verify that our model is valid.

Base R provides a built-in function to generate 4 critical diagnostic plots.

# Generate the 4-in-1 diagnostic plot
par(mfrow = c(2, 2)) # Set up a 2x2 grid
plot(model_mlr)
par(mfrow = c(1, 1)) # Reset grid
1. Residuals vs Fitted

Checks: Linearity and Homoscedasticity.

What to look for: The red dashed line should be roughly horizontal and flat around the zero line. If there is a curve (U-shape), the relationship is non-linear. If the spread of points widens or narrows (funnel shape), variance is unequal (heteroscedasticity).

2. Normal Q-Q

Checks: Normality of residuals.

What to look for: The points should closely follow the straight dashed diagonal line. If points deviate significantly at the tails (S-shape), the residuals are not normally distributed.

3. Scale-Location (Spread-Location)

Checks: Homoscedasticity (Equal Variance).

What to look for: Similar to plot 1, but uses square-root of standardized residuals. The red line should be horizontal. A rising trend indicates increasing variance.

4. Residuals vs Leverage

Checks: Influential observations.

What to look for: Identifies data points that disproportionately affect the regression line. Look for points falling outside the dashed red lines representing Cook's Distance.

Variance Inflation Factor (VIF)

VIF quantifies how much the variance of a regression coefficient is inflated due to multicollinearity (correlation among predictors).

$$ VIF_i = \frac{1}{1 - R_i^2} $$ Where $R_i^2$ is the R-squared from regressing predictor $X_i$ on all other predictors.
# Install and load the 'car' package
# install.packages("car")
library(car)

# Calculate VIF for the Multiple Linear Regression model
vif(model_mlr)

# Interpretation:
# VIF = 1: No correlation with other predictors.
# VIF < 5: Moderate correlation (usually acceptable).
# VIF > 5 (or > 10): High multicollinearity. 
# Action: Consider removing one of the highly correlated variables 
# or combining them (e.g., using PCA).
Cook's Distance & DFFITS

These metrics measure how much the model's coefficients change if a specific data point is removed. High values indicate "influential points" that might be skewing your results.

# Calculate Cook's Distance for all observations
cooks_d <- cooks.distance(model_mlr)

# Plot Cook's Distance
plot(cooks_d, type = "h", lwd = 3, col = "blue",
     main = "Cook's Distance", ylab = "Cook's Distance", xlab = "Observation Index")
abline(h = 4/nrow(mtcars), col = "red", lty = 2) # Threshold line

# Identify the row numbers of influential points
which(cooks_d > (4 / nrow(mtcars)))

# Alternative: DFFITS
dffits_vals <- dffits(model_mlr)
which(abs(dffits_vals) > (2 * sqrt(length(coef(model_mlr)) / nrow(mtcars))))

# Action: Investigate these specific rows in your dataset. 
# Are they data entry errors? Or are they genuine extreme cases? 
# If errors, remove them. If genuine, consider robust regression methods.

Binary Logistic Regression

When the dependent variable is categorical (specifically binary: Yes/No, 0/1, Success/Failure), linear regression fails because it can predict values outside the 0-1 probability range. Logistic regression solves this by using the logit link function to model the log-odds of the event occurring.

1. Simple Binary Logistic Regression

Concept & Equation

Instead of predicting $Y$ directly, we predict the probability $p$ that $Y=1$. To keep the output bounded between 0 and 1, we model the log-odds.

$$ \ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1X $$
  • $p$: Probability that the event occurs ($Y=1$).
  • $\frac{p}{1-p}$: Odds of the event occurring.
  • $\beta_1$: Change in the log-odds for a 1-unit increase in $X$.
Odds Ratios (OR)

To interpret the coefficients easily, we exponentiate them: $e^{\beta_1}$. This gives the Odds Ratio.

  • OR > 1: Increases the odds of the event.
  • OR < 1: Decreases the odds of the event.
  • OR = 1: No effect on the odds.
R Implementation (Using `mtcars` dataset)

The mtcars dataset has a binary variable vs (0 = V-shaped engine, 1 = Straight engine). We will predict engine shape based on miles per gallon (mpg).

# Load data
data(mtcars)

# Ensure the outcome is a factor (optional but good practice)
mtcars$vs <- as.factor(mtcars$vs)

# Fit Simple Logistic Regression using glm() with family = binomial
model_slog <- glm(vs ~ mpg, data = mtcars, family = binomial(link = "logit"))

# View summary
summary(model_slog)

# Calculate and interpret Odds Ratios
exp(coef(model_slog))
# If exp(mpg) = 1.2, it means for every 1 unit increase in mpg, 
# the odds of having a straight engine increase by 20%.

2. Multiple Binary Logistic Regression

Concept & Equation

Extends simple logistic regression to include multiple predictors. The interpretation of Odds Ratios remains the same, but now represents the effect of one predictor holding all other predictors constant.

$$ \ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1X_1 + \beta_2X_2 + \dots + \beta_pX_p $$
Core Assumptions
  • Binary Outcome: The dependent variable must be binary.
  • Independence: Observations are independent (no repeated measures).
  • Little/No Multicollinearity: Predictors are not highly correlated.
  • Linearity of Logits: Continuous predictors must have a linear relationship with the log-odds of the outcome (not the outcome itself).
R Implementation (Using `mtcars` dataset)

Predicting engine shape (vs) using miles per gallon (mpg), horsepower (hp), and weight (wt).

# Fit Multiple Logistic Regression
model_mlog <- glm(vs ~ mpg + hp + wt, data = mtcars, family = binomial)

# View summary
summary(model_mlog)

# Calculate Odds Ratios with 95% Confidence Intervals
exp(cbind(OR = coef(model_mlog), confint(model_mlog)))

# Predict probabilities for new data
new_cars <- data.frame(mpg = c(20, 30), hp = c(100, 150), wt = c(3.0, 2.5))
predict(model_mlog, newdata = new_cars, type = "response")
# 'type = "response"' returns probabilities (0 to 1) instead of log-odds.

3. Diagnostic Tests for Logistic Regression

Because logistic regression uses Maximum Likelihood Estimation (MLE) instead of Ordinary Least Squares (OLS), the diagnostics are different from linear regression.

1. Hosmer-Lemeshow Goodness-of-Fit Test

This test compares the observed event rates to expected event rates in subgroups of the model population (usually deciles).

  • Null Hypothesis ($H_0$): The model fits the data well.
  • Interpretation: A p-value > 0.05 indicates a good fit (we fail to reject $H_0$). *Note: This is the opposite of most statistical tests!*
# Install and load ResourceSelection package
# install.packages("ResourceSelection")
library(ResourceSelection)

# Perform Hosmer-Lemeshow test (requires g = 10 for deciles)
hoslem.test(model_mlog$y, fitted(model_mlog), g = 10)

# Check Pseudo R-Squared (McFadden's)
# install.packages("pscl")
library(pscl)
pR2(model_mlog)
# McFadden's R-squared between 0.2 and 0.4 indicates excellent model fit.
2. Linearity of Continuous Predictors and Log-Odds

Logistic regression assumes a linear relationship between continuous predictors and the log-odds of the outcome. We check this using the Box-Tidwell test.

# install.packages("boxTidwell")
library(boxTidwell)

# Test if 'mpg' and 'wt' have a linear relationship with log-odds of 'vs'
# Note: Box-Tidwell requires the outcome to be numeric (0/1), not factor
mtcars$vs_num <- as.numeric(as.character(mtcars$vs))

boxTidwell(vs_num ~ mpg + wt, data = mtcars)
# If p-value > 0.05, the linearity assumption is met.
ROC Curve and Area Under the Curve (AUC)

The Receiver Operating Characteristic (ROC) curve plots the True Positive Rate (Sensitivity) against the False Positive Rate (1 - Specificity) at various threshold settings. The AUC provides a single number summarizing the model's discriminatory power.

  • AUC = 0.5: No discrimination (random guessing).
  • 0.7 ≤ AUC < 0.8: Acceptable discrimination.
  • 0.8 ≤ AUC < 0.9: Excellent discrimination.
  • AUC ≥ 0.9: Outstanding discrimination.
# Install and load pROC package
# install.packages("pROC")
library(pROC)

# Calculate ROC object
# Direction="<" means higher predicted values indicate the event (1)
roc_obj <- roc(mtcars$vs_num, fitted(model_mlog), direction = "<")

# Plot the ROC Curve
plot(roc_obj, 
     main = "ROC Curve for Logistic Regression",
     col = "blue", 
     lwd = 2, 
     print.auc = TRUE) # Prints the AUC value on the plot

# Get the exact AUC value
auc(roc_obj)
Confusion Matrix & Classification Metrics

To evaluate how well the model predicts actual categories, we convert the predicted probabilities into classes (0 or 1) using a cutoff (usually 0.5) and compare them to the actual values.

# Predict probabilities
probs <- fitted(model_mlog)

# Convert to classes using 0.5 threshold
predicted_classes <- ifelse(probs > 0.5, 1, 0)

# Create Confusion Matrix
# install.packages("caret")
library(caret)

actual_classes <- mtcars$vs_num
confusionMatrix(as.factor(predicted_classes), as.factor(actual_classes), positive = "1")

# Key Metrics Explained:
# Accuracy: Overall correctness of the model.
# Sensitivity (Recall): True Positive Rate (How well it predicts '1's).
# Specificity: True Negative Rate (How well it predicts '0's).
# Precision: When it predicts '1', how often is it actually '1'?
Pro Tip: If your data is imbalanced (e.g., 90% 0s and 10% 1s), accuracy is a misleading metric. Focus instead on Sensitivity, Precision, and the F1-Score (the harmonic mean of Precision and Recall).
